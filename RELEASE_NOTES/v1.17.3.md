# MacCrab v1.17.3 — notification, navigation, and update-resilience fixes

A maintenance release: two navigation/notification fixes, a rule-search
improvement, and resilience hardening for the update process.

## Fixed

- **Tapping a MacCrab notification now opens the alert.** Clicking a
  notification banner reliably navigates to that alert in the dashboard
  (previously it could report that the link couldn't be opened).
- **"Open rule" on a multi-step detection** — a sequence or trace-graph
  rule — now shows an explanation instead of an empty rule list, matching
  the behavior for MacCrab's other built-in detections.

## Improved

- **Search rules by description.** The Detection rule search now also
  matches rule descriptions, in addition to the rule id, name, category,
  and MITRE technique.
- **Smoother updates.** During an update the app no longer briefly
  double-posts notifications while the previous detection engine is handed
  over, and the dashboard recovers on its own if it connected before the
  updated engine finished starting — no manual restart needed.

## Upgrade

- **Homebrew:** `brew upgrade --cask maccrab`
- **Sparkle:** accept the in-app update prompt and relaunch when asked.
- **Manual:** download the DMG from the release page and replace MacCrab.app.

Universal (Apple silicon + Intel), Developer ID signed and notarized.
Minimum macOS 13.
