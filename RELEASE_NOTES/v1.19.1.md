# MacCrab v1.19.1

A forensic-insight, parity, and polish release. The detection engine already computes rich, structured findings â€” this release makes sure that detail actually reaches you, in whichever interface you use, and lets every interface drive the core security operations.

## Forensic insight reaches you

- **Findings carry their full detail everywhere.** Severity, the plain-language explanation, the backing evidence, and the computed TCC risk score now flow through the MCP tools, the dashboard, and the CLI â€” instead of being collapsed to a one-line summary.
- **The dashboard reflects the engine's real severity.** A critical posture finding now shows as critical (not a flat "needs review"), and TCC grants are graded by the engine's risk model.
- **New CLI views into a scan:** `maccrabctl scan findings`, `scan explain`, and `scan timeline`.
- **MITRE ATT&CK and D3FEND references are humanized and clickable** on alerts.

## Every interface can drive MacCrab

- **Response actions** are now configurable from the CLI (`maccrabctl actions`) and from AI agents over MCP (`list_response_actions` / `set_response_action`) â€” previously dashboard-only. Destructive actions (kill / quarantine / network-block) default to confirmation-gated, and agent access is capability-gated.
- **New CLI commands for parity with the MCP tools:** `package` (supply-chain analysis), `ai-alerts`, `scan-text` (prompt-injection scan), `config get/set`, `rule delete` / `rule severity`, `session` (signed agent-session export & verify), and `evidence search` / `evidence show`.
- **Forensic enrichers and analyzers** are now runnable over MCP (`forensics_enrich`, `forensics_run_analyzer`) and the CLI â€” they were previously reachable from no interface.

## Compatibility & polish

- **MCP forensic tools are underscore-named** (`forensics_run_collector`, â€¦) so strict MCP clients accept them; the legacy dotted names still work as aliases.
- **Eight high-traffic dashboard surfaces are now fully localizable.**
- **AI-Guard "tools observed"** is a real by-tool table, and **Prevention** shows live sinkhole / network-blocker / persistence-guard status.

## Correctness & safety

- Alert counts are exact (no longer capped or undercounted on busy hosts).
- Response actions set via any interface now reliably take effect.
- File-analysis collectors are size-bounded, hash files by streaming, and never follow symlinks.
- Plugin uninstall is confirmation-gated; forensic tool error messages are sanitized; daemon status reports liveness from heartbeat recency, not a leftover write-ahead-log file.

All MCP responses remain privacy-sanitized, all LLM features stay advisory, and the local-first model is unchanged.
