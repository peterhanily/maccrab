# v1.13.0-rc.1 — Mac Context Plugin Platform substrate (sub-slice 1)

**Status: internal release candidate.** Not published to Sparkle appcast or Homebrew cask. Source tag only.

This RC closes v1.13a-1, the foundational sub-slice of the four-release Mac Context Plugin Platform arc (v1.13a → v1.13b → v1.14 → v1.15). The substrate now exists end-to-end, but ships no real collectors yet — those start landing in v1.13.0-rc.2 (TCC-lite arrives in sub-slice 3, after the codesign-resolve enricher in sub-slice 2).

## What landed

**New SPM target.** `MacCrabForensics` — the Mac Context Plugin Platform library. Hosts the plugin runtime, encrypted ArtifactStore, Case model, BlobVault, plugin registry, and the fixture plugin. Linked into maccrabctl + MacCrabApp + maccrab-mcp; intentionally NOT linked into MacCrabAgent (the ES system extension) or maccrabd (the legacy daemon) so a plugin bug can't crash the detection engine.

**Vendored SQLCipher.** The macOS-bundled libsqlite3 is no longer in the dependency tree. The codebase now links a single SQLite implementation: SQLCipher 4.16.0 (SQLite 3.53.1) compiled into a vendored `CSQLCipher` SPM target with the CommonCrypto crypto backend. Existing un-encrypted stores (events.db, alerts.db, etc.) continue to work transparently because SQLCipher with no PRAGMA key behaves identically to upstream SQLite. Encryption is now available on demand for any store.

One side-effect: SQLite ≥3.53 no longer auto-checkpoints inside VACUUM, so `EventStore.vacuum()` now sandwich-checkpoints — drain the WAL before VACUUM, drain again after. This restores the size-cap shrink contract that the v1.6.12 prune+vacuum test asserts.

**Plugin protocol surface.** `ForensicPlugin` base protocol plus four kind-specific sub-protocols (`Collector`, `Enricher`, `Fingerprinter`, `Analyzer`), each with its own invocation signature. `PluginManifest` carries id / version / type / runtime / tccRequirements / inputs / outputs / mcpTools / schemaVersion / stability. v1.13a accepts `runtime = .tierA` only; `.tierB` is reserved for the post-store sandboxed-subprocess runtime. The five-class privacy model (`metadata` / `content` / `personalComms` / `credentialAdjacent` / `secret`) binds the whole platform.

**Encrypted ArtifactStore.** SQLCipher-backed `case.sqlite` per case. Schema v1 carries `cases` / `artifacts` / `artifact_data` / `plugin_invocations` with `observed_at` distinct from `captured_at` (timelines reflect "when did this happen," not "when did we look"). The store is the single writer for those tables — audit Pass 2026-B enforces by source-tree scan.

**Pass 2026-D plaintext-rejects-non-metadata at INSERT.** `ArtifactStore.commit(_:)` reads `encryption_state` at construction and refuses any non-metadata-class artifact when the case is plaintext. Six dedicated tests prove the gate for each non-metadata class.

**Case model with login-keychain DEK wrap.** Each case carries its own random 32-byte AES-256 DEK, wrapped via `KeychainDEKVault` as a generic-password Keychain item with `kSecAttrAccessControl` bound to `.devicePasscode` (Touch ID convenience arrives in v1.13b). DEKs never escape to iCloud Keychain. `InMemoryDEKVault` covers the test path so CI never prompts for auth.

**BlobVault** (AES-GCM per-blob with sha256-derived nonces) for the large-artifact spill path. The nonce derivation gives the storage layer the dedup property the plan asks for: same content + same DEK → byte-identical ciphertext.

**maccrabctl `case` + `plugin` subcommands.** Operator-facing CLI for case lifecycle (`new` / `list` / `show` / `artifacts` / `allow-ai` / `mark-trusted-scheduled` / `delete --shred`) and plugin runtime (`list` / `info` / `run`). Verified end-to-end against the fixture plugin.

**Fixture plugin** (`com.maccrab.forensics.fixture`). No-op Collector that emits a single `fixture.heartbeat` artifact. Exercises the platform's full lifecycle without depending on macOS TCC.db or launchd plists.

**Two new audit passes** in `scripts/pre-release-audit.sh`:
- **Pass 2026-A** — plugin manifest integrity. Every plugin manifest in the source tree must be registered in `Bootstrap.registerBuiltins`; first-party ids must live under the reserved `com.maccrab.{forensics,enricher,fingerprinter,analyzer}.*` namespaces; every output declares `privacyClass`.
- **Pass 2026-B** — single ArtifactStore writer. No code outside `Sources/MacCrabForensics/Storage/ArtifactStore.swift` may `INSERT INTO` the `artifacts` / `artifact_data` / `plugin_invocations` tables.

## What's intentionally NOT in this RC

Per plan §9.5 scope-discipline guardrails — every item below stays in backlog until the v1.15 posture Analyzer demonstrates the platform produces findings from artifacts:

- TCC-lite, launchd-lite, codesign-resolve enricher (land in v1.13a-2 through v1.13a-5).
- Dashboard Forensics tab (v1.13b).
- Touch ID convenience layer (v1.13b).
- Scheduled-run consent UX (v1.13b).
- MCP meta-tools (`forensics.list_plugins` / `forensics.run_collector` / `forensics.search_artifacts` / `forensics.get_artifact`) — wiring lands with v1.13a-3.
- MCFP, posture Analyzer, AppleScript runtime monitor — all post-v1.13a.
