# MacCrab v1.9.0 — Agent Traces

The biggest feature minor since the LLM backend integration.
v1.9.0 introduces **Agent Traces** — a correlation pathway between AI
coding-agent activity and the kernel events MacCrab already observes.
For every kernel event under an AI agent, MacCrab can now answer
*"which agent interaction caused this?"* — bound by W3C trace IDs when
the agent emits OTel telemetry, by process lineage when it doesn't.

This is the kind of feature that has been impossible on macOS until
now: AgentSight (arXiv 2508.02736) demonstrated the architecture on
Linux via eBPF; v1.9.0 brings it to macOS via Endpoint Security +
W3C TRACEPARENT propagation.

## What's new

### Agent Traces (the headline feature)

- **TRACEPARENT propagation at execve.** When Claude Code (or any
  OTel-instrumented AI tool) spawns a subprocess, MacCrab reads the
  inherited W3C trace context from the env block at NOTIFY_EXEC time
  and binds it to the process. Every subsequent kernel event from
  that process and its descendants carries the originating
  `trace_id` / `span_id`.
- **Lineage fallback.** When TRACEPARENT isn't propagated (Cursor,
  Copilot CLI, Codex without telemetry), MacCrab attributes via
  process ancestry against `AIToolRegistry`. Lower confidence,
  surfaced in alerts as `machine_agent_confidence: lineage`.
- **Loopback OTLP receiver.** Optional in-daemon HTTP receiver bound
  to `127.0.0.1:4318` ingests OTel traces from AI tools running
  alongside MacCrab. Spans land in a separate `traces.db`.
- **Wire-boundary sanitiser.** Every span attribute passes through a
  secret-shape redactor before persistence — Anthropic / OpenAI /
  Google / AWS / GitHub / Slack / Stripe (incl. webhook signing) /
  npm / Twilio (key + SID) / JWT / Postman / SendGrid / Mailgun /
  Discord webhook URL / Cloudflare / DigitalOcean / Heroku / Vercel.
  Plus a Shannon-entropy fallback that catches unknown-vendor secrets
  (≥40 char base62 with entropy ≥4.5 bits/char) while sparing
  pure-hex hashes.
- **AES-GCM at rest.** `attributes_json` is column-encrypted under the
  same shared key as `events.db` / `alerts.db`. Tamper detection via
  GCM authentication tag.
- **Three new detection rules** (all `stable`):
  - `agent_filesystem_violation_high_conf` — high-severity:
    TRACEPARENT-bound agent writes to `/Library/LaunchDaemons/`,
    `/Library/LaunchAgents/`, `~/.ssh/`, `~/Library/Keychains/`.
  - `agent_filesystem_violation_probable` — medium-severity:
    same paths but lineage attribution only.
  - `agent_traceparent_credential_access` — high-severity:
    TRACEPARENT-bound agent reads SSH keys / AWS creds / cloud SDK
    tokens / browser-extension wallet directories.

### Dashboard

- **New "Agent Traces" panel** (Intelligence sidebar group). Lists
  recent traces, renders span hierarchy with parent/child indent and
  pretty-printed attributes JSON. Stats bar separates OTLP-ingested
  spans from machine-attributed events.
- **In-panel receiver toggle** with live status pill. No env vars
  required for the common path — flip the switch, daemon SIGHUPs,
  receiver starts/stops cleanly.
- **"Originating Agent Trace" box** in alert detail — shows
  trace_id/span_id/agent/confidence, plus a thumbs UI for the
  operator's verdict overlay (Confirm / Wrong tool / No agent /
  Note). Machine attribution is immutable; verdicts live separately
  in `attribution_overrides.db`.
- **"Show in Agent Traces"** cross-link from alert detail to the
  Agent Traces panel.

### Settings

- **"Manual event prune"** — operator-triggered immediate
  size-cap sweep + VACUUM on events.db. Sends SIGUSR2 (and SIGHUP
  fallback for older daemons). Surfaces current size, last-run
  timestamp, before→after delta. Workaround for hosts where the
  hourly enforcer hasn't kept up.
- **"System Extension"** — clean removal path via
  `OSSystemExtensionRequest.deactivationRequest`. Doesn't require
  disabling SIP. Surfaces sysext state with a colored pill.
- **AI Activity timeline** — replaced horizontal session chip strip
  with a Menu-style picker plus inline detail bar; counts are
  pluralized correctly and surface the alert-bearing-session count
  alongside the total.

### CLI

- **`maccrabctl status`** gains an Agent Traces line with the
  canonical `attribution_accuracy_among_rated: <pct> (rated=N,
  total=M)` quality-metric format.
- The status line resolves DB paths against both system + user dirs
  so a stale `/Library/.../events.db` from a prior root daemon run
  doesn't mask the running non-root daemon's data.

### Threat intel hardening

- **Multi-tenant platform suffix guard** — `pages.dev` /
  `vercel.app` / `firebaseapp.com` / `replit.app` / `glitch.me` /
  `r2.dev` / etc. are excluded from the domain suffix walk. A bad
  `attacker.pages.dev` IOC no longer blanket-flags every
  `*.pages.dev` site.
- **Anchored URL match** — `isURLMalicious` and `recordForURL` now
  require exact match or `hasPrefix`, not arbitrary substring.
- **`MACCRAB_ABUSECH_AUTH_KEY`** env var support — when set, sent as
  the `Auth-Key` header on abuse.ch feed fetches. Raises the
  rate-limit ceiling for fleet deployments behind a single egress IP.
- **`staleFeeds(staleMultiplier:)`** API for surfacing feeds that
  haven't refreshed in 2× the update interval.
- **`loadCustomIOCFiles()`** now actually called from `start()` —
  drop-in `*.hashes.txt` / `*.ips.txt` / `*.domains.txt` files are
  loaded at boot (was dead code prior to v1.9.0).

## Performance hardening

- **OTLP body accumulator O(N²) → O(N)**: 8 MB body in 64 KB chunks
  was ~512 MB of memory traffic; class-wrapped buffer drops it to
  ~8 MB.
- **MinimalProtoReader on `Data` slices** (no `Array(data)` per
  nested decoder) — ~5× allocation reduction per request.
- **Slow-loris timeout** (10 s) per OTLP connection.
- **Hard `maxBodyBytes` enforcement** during accumulation, even if
  Content-Length lies.
- **`autoreleasepool`** around the receiver hot decode path.
- **Shared `AIToolRegistry`** instance — was allocated per
  NOTIFY_EXEC; now once per daemon lifetime.
- **`refreshAgentTraces` real mtime-skip** — the dashboard no
  longer hits SQLite every 5 s when nothing has changed.

## Stability

- **`traceBindings` AsyncStream bounded** (`bufferingNewest(10_000)`)
  — no unbounded growth under exec bursts.
- **TraceStore handle leak fix** on prepare-failure.
- **Path-probe on EventStore + AlertStore + TraceStore reads** —
  picks whichever DB has the freshest mtime so dashboard stays in
  sync regardless of which daemon (root sysext vs non-root dev)
  is active.
- **SIGTERM clean shutdown** stops the OTLP receiver before the
  rest of the daemon — releases :4318 cleanly so the next launch
  doesn't hit TIME_WAIT.
- **`SecurityScorer.isProcessRunning` rewritten** to avoid the
  Foundation `terminationStatus` NSException crash class (was
  pre-existing, surfaced under heavy dashboard refresh load).

## Audit invariants codified

Pass 12 — `traces.db` opener count = 1.
Pass 13 — env-block accessors confined to TraceExtractor pathway.
Pass 14 — every YAML-rule-referenced enrichment key has a Swift producer.
Pass 15 — daemon-target `TraceStore(directory:)` calls always pass
`encryption:`. Adding any of these without the wiring fails
`pre-release-audit.sh`.

## Compatibility

- macOS 13+ (unchanged).
- Schema migrations on `events.db`: v4 (agent_* columns) + v5 (legacy
  `attribution_overrides` table; functionally replaced by the
  user-writable `attribution_overrides.db`). Existing v1.8.1
  installations migrate automatically; idempotent re-runs handled by
  the SchemaMigrator pattern shipped in v1.7.6.
- `traces.db` is brand-new; created on first OTLP-receiver activation.
- Rule corpus: 427 (was 425).

## Upgrading

`brew upgrade --cask peterhanily/maccrab/maccrab` or download the DMG
from the GitHub release. Sparkle in-app updates roll out behind the
appcast as usual. After upgrade the System Extension activator picks
up the new sysext bundle on next launch; OS approval prompt may
appear if the bundle ID team prefix changed.

If you have a v1.8.x sysext that's bloated `events.db` past 1 GB, the
new **Settings → Manual event prune** button runs the size-cap
enforcer + VACUUM immediately. Or use **Settings → System Extension →
Remove System Extension** to uninstall the old one cleanly without
disabling SIP.

## Known follow-ups (v1.9.1)

- Underlying v1.8.x `events.db` size-cap regression (cause of the
  multi-GB growth before the manual flush).
- `PromptInjectionScanner.isAvailable` Process-launch failure path —
  same `try? proc.run(); terminationStatus` shape as the
  SecurityScorer fix; not currently triggered because forensicate is
  optional.
- Manifest decode warning at rule-load (`keyNotFound CodingKeys("id")`).
- `traceStoreOrNil` cache invalidation across user/system path flips.
- Settings UI for OTLP receiver port (currently 4318 only).
