## MacCrab v1.7.0 — MCP attribution + behavioral baseline (producer half)

The first feature minor since v1.6.0. Closes the longest-standing
"wire-the-orphans" gap in the codebase: `MCPBaselineService` has
existed in `Sources/MacCrabCore/AIGuard/` since v1.6.6 with a complete
learning/enforcing API and `BaselineDeviation` AsyncStream — but the
**producer half** that actually feeds it observations from real
events was never built. v1.7.0 builds it.

### What's new

**Per-event MCP attribution.** Every event MacCrab observes from a
process descended from an AI coding tool (Claude Code, Cursor,
Continue.dev, VS Code, Windsurf, Aider, Codex) is now walked against
the AI tool's configured MCP servers. When an ancestor process
matches one (cmdline contains the package token like
`@modelcontextprotocol/server-filesystem` / `mcp-server-fetch` /
`firecrawl-mcp`), the event is tagged with:

- `mcp_server_name` — the configured server name (e.g. `filesystem`,
  `github`, `brave-search`)
- `mcp_server_category` — derived from the package token
  (`filesystem`, `github`, `fetch`, `memory`, etc.)
- `mcp_attribution_confidence` — `high` (package-token match),
  `medium` (executable basename + arg match), or `low`

Every high/medium-confidence attributed event feeds
`MCPBaselineService.observe(...)`, which builds a per-(tool, server)
fingerprint of file basenames, network domains, and child process
basenames. After 20 observations spanning at least 5 minutes, the
baseline promotes from `learning` to `enforcing`. From then on, any
observation outside the fingerprint produces a Baseline Drift alert:

```
maccrab.mcp.baseline-anomaly.<tool>.<serverName>.<kind>
```

severity `medium`, MITRE-mapped to `attack.initial_access` /
`attack.command_and_control`. Routed through the existing AlertSink
chokepoint so dedup, suppression, and feedback-rate behavior all work
out of the box.

**MCP Server Activity panel** in the dashboard's Intelligence group
(after AI Analysis). Per-server rows show the configured server
name, AI tool host, observation count, and learning/enforcing badge.
Click a row to inspect the full fingerprint — every file basename,
domain, and child-process basename the baseline has learned. A
banner at the top surfaces recent Baseline Drift alerts as a
shortcut to the Alerts panel.

### How it's wired

```
ES event ──► EventLoop enricher chain ─► AI-child detection
                                          │
                                          ▼
                                  MCPAttributor.attribute()
                                          │
                                  ┌───────┴───────┐
                                  │               │
                                 nil          Attribution
                                  │               │
                                  │      enrichments[mcp_*]
                                  │               │
                                  │       MCPBaselineService.observe()
                                  │               │
                                  │      learning → enforcing
                                  │               │
                                  │      BaselineDeviation stream
                                  │               │
                                  │         AlertSink.submit
                                  ▼               ▼
                              continue        notification
```

`MCPAttributor` (new actor in `Sources/MacCrabCore/AIGuard/`) caches
attribution by PID with a 5000-entry LRU. Negative results are
cached too — events from non-MCP processes pay one walk and zero
re-walks. Hot path is O(1) on a cached PID.

`MCPBaselineService.writeSnapshot(to:)` (new) serializes every
learned baseline to `<supportDir>/mcp_baselines.json` on the 30 s
heartbeat tick. The dashboard reads it via
`MCPBaselineService.readSnapshot(at:)` (also new) into
`AppState.mcpBaselines`. Same atomic temp+rename pattern as
`AgentLineageService.writeSnapshot`.

### Notes for operators

- Baselines need **both** 20 observations and 5 minutes of wall-clock
  before they enforce. A single afternoon of agent activity will
  promote 1–3 servers to enforcing; the rest stay learning until
  more data accrues.
- Baseline drift alerts default to `medium` severity. Adjust per-rule
  in Settings → Response Actions if you want them louder/quieter.
- Domains are normalized to their registrable form before
  fingerprinting (`api.github.com` → `github.com`); subdomains aren't
  treated as drift unless a wholly-new registrable domain appears.
- `<supportDir>/mcp_baselines.json` is world-readable (0o644) so the
  dashboard can decode it under user privilege. Contents are
  fingerprint sets, not raw event data.
