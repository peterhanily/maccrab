# v1.16.0-rc.7 — FSEvents discovery + Mach-O analyzer

Two plugins. Closes §13.6 (FSEvents at discovery-level — binary record-stream parsing deferred) and opens §13.7 file analyzers with Mach-O.

## What landed

**`com.maccrab.forensics.fsevents`** — discovery plugin. Enumerates `/.fseventsd/` log files (gzipped binary records) with filename + size + mtime. Each emits an `fsevents.log_file` artifact (metadata class). Full binary record-stream parsing deferred — the gzip + variable-record stream needs a focused parser; this RC ships the discovery surface so operators can see when FSEvents was truncated + how much history is on disk.

**`com.maccrab.forensics.macho-analyzer`** — first §13.7 file analyzer. Reads a Mach-O binary via `otool -L` for dylib deps + `codesign -d --entitlements :-` for entitlement keys + Mach-O magic byte read for arch + existing `CodeSigningCache` for team_id / signing status. Emits one `macho.analysis` artifact per target. Dogfood default set (`/usr/bin/true`, `osascript`, `codesign`) until PluginRunner threads operator-supplied inputs through (follow-up).
