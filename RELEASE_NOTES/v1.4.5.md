## MacCrab v1.4.5 — Wave B: five field-data noise kills

Five noise sources observed during v1.4.4 field testing on a real
MacBook Pro developer workstation get shut down. Nothing in this
release adds detection — every change removes known FPs.

### Fixed

- **Power-anomaly alerts on Apple system daemons.** `useractivityd`
  holds a BTLE advertisement wake lock when Handoff, AirDrop, or
  other continuity features are active. `appleh13camerad` holds
  one during every video capture. Same for `applecamerad`,
  `nsurlsessiond`, `mobileassetd`, `assetsubscriptiond`,
  `AssetCacheLocatorService`, `AssetCache`. All added to the
  `PowerAnomalyDetector.knownLegitimate` allowlist.

- **Power-anomaly alerts on Signal / meeting apps.** Signal's
  "Video Wake Lock" during a call was firing at Medium. Added
  Signal, Google Meet, Zoom, Cisco Webex to the allowlist
  (Zoom.us, FaceTime, Slack, Teams, Discord were already in).

- **CryptoTokenKit extension alerts no longer fire HIGH on
  legitimate auth hardware.** Plugging in a YubiKey or installing
  1Password added a Non-Apple CTK extension — reported at HIGH
  severity. Now: known-signed CTK providers (Yubico, 1Password,
  OneSpan, Thales, Entrust, Gemalto, OpenSC, mTrust) are skipped
  entirely, and unknown CTK extensions drop from HIGH to
  Informational. The sysext still records the enumeration.

- **USB rate limiting.** Every hub / HID / YubiKey connect surfaced
  an informational alert on every plug-in. New `USBRateLimiter`
  actor tracks (vendor_id, product_id, direction) tuples and
  skips duplicates within a 24-hour window per sysext session.
  Mass-storage events bypass the limiter (exfil-class, every one
  matters).

- **Behavioral Score Threshold on trusted browser helpers.**
  Google Chrome Helper was accumulating score from
  `sigma_rule_match_critical` contributions that were themselves
  FPs on known-noisy rules — firing HIGH "Behavioral Score
  Threshold" alerts on every Chrome session. Browser paths
  (Chrome / Safari / Edge / Firefox / Arc / Electron app helpers
  per `NoiseFilter.isTrustedBrowserHelper`) no longer contribute
  to the behavioral score at all.

- **Gate 5 now recognizes shell parents as interactive.** v1.4.0
  Gate 5 required a desktop-terminal ancestor (Terminal.app,
  iTerm, Warp, etc.) to suppress admin-CLI discovery alerts. On
  some machines the ES ancestor chain stopped at the shell
  binary (`/bin/zsh`) without reaching the Terminal.app
  executable. Now: a shell basename (bash, zsh, sh, fish, dash,
  ksh, tcsh, csh) as an ancestor is enough. Covers the "ps
  fired by `ps` from zsh" case directly observed in field data.

- **SSH Agent Socket rule excludes shell parents.** `ssh_agent_access_suspicious`
  was firing at HIGH on Python doing legitimate Paramiko / Ansible
  / Fabric work from a developer's terminal. New `filter_terminal`
  excludes parents of `/bash`, `/zsh`, `/sh`, `/fish`, `/dash`.
  Attackers don't chain SSH-agent hijack through Terminal + zsh;
  they do it from a dropped binary or a launchd-spawned script.

---

No schema changes. No new health signals. 600 tests pass (pre-existing
`LLM Cache` flake unrelated to this release).
