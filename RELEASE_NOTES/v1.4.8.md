## MacCrab v1.4.8 — PlatformBinary filter, hub + CTK noise kills

Another field-test pass on v1.4.7 surfaced the same four
discovery rules still firing on Apple's own command-line tools,
plus two more noise sources (third-party USB hubs, pluginkit
error lines). This release adds a single-line filter primitive
that fixes the discovery-rule class systemically.

### Added

- **`PlatformBinary` Sigma field.** New rule-level selector
  that resolves directly to `process.is_platform_binary` — the
  ES-framework-provided bit that distinguishes Apple platform
  binaries from everything else. Unlike `SignerType`, it comes
  straight from the kernel event and is never nil. Mapped in
  `Compiler/compile_rules.py`. Use `PlatformBinary: 'true'`
  in a filter section to exclude Apple platform software.

### Fixed

- **Discovery rules firing on Apple CLI tools from shell
  parents.** `system_enumeration_burst`, `xpc_service_enumeration`,
  `csrutil_status_check`, `process_listing_by_unsigned`, and
  `defaults_read_sensitive` all now carry a `filter_platform:
  PlatformBinary: 'true'` step. Root cause: the existing
  `filter_apple: SignerType: 'apple'` silently failed when the
  `SecStaticCode` enrichment returned nil for short-lived
  Apple CLI tools (launchctl, system_profiler, etc.) — rule
  engine treats a nil field as "predicate false", so
  `not filter_apple` evaluated `true` and the rule fired.
  PlatformBinary doesn't depend on enrichment and closes the
  hole.

- **USB hubs no longer generate informational alerts.** Third-
  party USB-C docks, 4-port splitters, and monitor-integrated
  hubs (Realtek 0x0bda, VIA 0x2109, Intel 0x8087) churn
  connect/disconnect on every replug and every USB-C mode
  renegotiation. USB device class `0x09` (hub) is now skipped
  upstream of the rate limiter, same policy as Apple VID 0x5AC.
  Mass-storage (class 0x08) still surfaces regardless of
  vendor — exfil class.

- **pluginkit error lines treated as CTK plugins.** On machines
  with smartcard daemons in transitional states `pluginkit -m`
  emits status rows like `match: Connection invalid` that were
  being parsed as CTK plugin bundle IDs and fired as "Non-Apple
  CryptoTokenKit extension" alerts. The SystemPolicyMonitor now
  skips lines containing `Connection invalid`, `Operation not
  permitted`, `No such`, or `error`.

600 tests pass. No schema changes.
