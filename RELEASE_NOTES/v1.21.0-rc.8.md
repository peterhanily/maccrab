# MacCrab v1.21.0-rc.8

A correctness pass from a deep-dive audit, fixing a false "Protection degraded" banner and several scoring/UX accuracy issues.

## Fixes

- **No more false "Protection degraded".** After an install/update the dashboard could latch a "no rules loaded" reading and show *Protection degraded* for the whole session even though the engine was healthy. The app now re-checks until rules load, falls back to the system rules directory, and the banner reflects real engine health — a low security score (e.g. firewall off) is shown by the Security Grade tile instead of crying wolf about the engine.
- **Honest security score.** XProtect is Apple-managed, so it's no longer false-flagged as "needs updating"; the Screen Lock check no longer false-warns on modern macOS where the setting key is absent.
- **Accurate rule count** on the welcome screen (shows the rules actually enforced).
- **MCP hardening.** The agent plugin-install tool can't be steered to a local sideload, and the capability-gate test now covers every plugin mutator.
- **Crabby polish.** VoiceOver labels on his buttons; his animation pauses when the window isn't active.

## Carried forward (rc.1 → rc.7)

- Masonry Overview (no gaps), the rave pixel-crab Crabby with props/animations/interactions, Protection Coverage + Top Firing Rules widgets, catalog Update/Reinstall/Remove, Forensics CLI↔MCP parity, and the must-fire suppression carve-out.
