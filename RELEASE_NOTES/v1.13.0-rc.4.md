# v1.13.0-rc.4 — launchd-lite collector + plist parser

**Status: internal release candidate.** Not published to Sparkle appcast or Homebrew cask. Source tag only.

Closes the second-to-last sub-slice of v1.13a. The platform now answers plan §9.5.1's second v1.13 question: *what persistence exists on this Mac, and what is unsigned, unfamiliar-team-signed, or recently modified?*

## What landed

**`com.maccrab.forensics.launchd-lite`** — Collector that walks every standard launchd plist directory:

- `/Library/LaunchAgents` (system-wide-agent)
- `/Library/LaunchDaemons` (system-wide-daemon)
- `~/Library/LaunchAgents` (user-agent)
- `~/Library/StartupItems` (legacy-startup)
- `/Library/StartupItems` (legacy-startup)

Each plist is parsed via `LaunchdPlistParser`, then enriched with codesign posture via `CodesignResolveEnricher` so `codesign.team_id` + `codesign.signing_status` + `codesign.notarized` + `codesign.hardened_runtime` land alongside the launchd metadata on the same `launchd.entry` artifact. Sigma rules and dashboard panes that want "unsigned launchd entries" can match in one query.

Per-entry fields per plan §4.2:

- `domain`, `plist_path`, `label`
- `program_path`, `arguments_json`, `arguments_count`
- `run_at_load`, `keep_alive`, `start_interval_seconds`, `watch_paths_json`, `process_type`
- `runs_as_root`, `effective_user`, `source_user`
- `program_exists`, `program_missing_reason` (`deleted` / `inaccessible` / `unresolved`)
- `plist_mtime_ms`
- `codesign.*` (from the enricher cross-reference)

**LaunchdPlistParser.** Pure-function parser exercised directly by tests. PropertyListSerialization for XML + binary plist formats. Handles `Program` (single) vs `ProgramArguments` (array); decodes `KeepAlive` as either Bool or condition dictionary; respects `UserName` key override (a system-daemon with `UserName=_spotlight` runs as `_spotlight`, not root, per launchd's documented behavior). Missing or unreadable program paths produce `program_exists=false` with a structured `program_missing_reason`.

## Tests

1745 total across 323 suites passing. v1.13a-4 additions:

- `LaunchdPlistParser` — 10 cases covering Label + ProgramArguments parsing, system-daemon → runs_as_root, UserName key override, KeepAlive dictionary form, StartInterval preservation, missing-program reasons (deleted / unresolved), WatchPaths round-trip, missing-Label error, invalid-XML error.

## What's intentionally NOT in this RC

- **BAM (BackgroundItems-v9.btm) parser** — emits a coverage-advisory note instead. The format is partially documented binary; full parse lands in a v1.13a-4 follow-up. The plist-side persistence coverage is the dominant macOS surface.
- **`/System/Library/Launch*` baseline** — manifest exposes `includeSystemBaseline` but PluginRunner doesn't yet thread per-invocation inputs through to plugins. v1.13a-1 follow-up.
- **TCC-lite ↔ codesign-resolve cross-reference** — TCC.db's `client` is often a bundle id, not a path, so cross-reference needs `LSCopyApplicationURLsForBundleIdentifier` resolution first. Deferred.
- **overrides.plist enabled/disabled flag** — per-domain `overrides.plist` lookup not yet wired. Deferred.
