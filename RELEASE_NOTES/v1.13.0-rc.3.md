# v1.13.0-rc.3 — TCC-lite collector + LiveDBSnapshot + risk scoring

**Status: internal release candidate.** Not published to Sparkle appcast or Homebrew cask. Source tag only.

Closes v1.13a-3 of the Mac Context Plugin Platform arc. The platform now answers the first of plan §9.5.1's three v1.13 questions: *what apps have FDA / Accessibility / Automation / Screen Recording / Mic / Camera / AppleEvents?*

## What landed

**`com.maccrab.forensics.tcc-lite`** — first concrete Collector. Snapshots `~/Library/Application Support/com.apple.TCC/TCC.db` (user scope) and `/Library/Application Support/com.apple.TCC/TCC.db` (system scope) before parsing, then iterates the `access` table and emits:

- One `tcc.grant` artifact per access row, carrying canonical `service` + `service_raw` + `client` + `client_type` + `auth_value` + `auth_reason` + `indirect_target` (Automation grants) + `csreq_hash` + `risk_score` + `risk_reason`.
- One `tcc.summary_by_service` aggregate per service: `count`, `allowed`, `denied`, `risk_max`.

All artifacts carry privacy class `metadata` — exposable to MCP without operator opt-in (the plan binds the substrate to metadata-only).

**LiveDBSnapshot.** Atomically copies an actively-written SQLite database into the case's `snapshots/<sha256>.db` via `sqlite3_backup_init/step/finish`. The backup API drains the WAL into the destination so the resulting file is internally consistent even when tccd is writing in the background. Reusable by future collectors (launchd-lite's BAM snapshot lands in v1.13a-4).

**Deterministic TCC risk scoring.** `TCCRiskScoring.score(_:)` is a pure function of the grant's service / target / auth signals. Every weight is named (`Weight.fullDiskAccess = 35`, `Weight.accessibility = 30`, etc.) so the audit + operator can argue with the numbers, not the concept. Reasons are surfaced in stable order via the `TCCRiskReason` enum. Clamped to `[0, 100]`.

**Service normalization.** `TCCServiceNormalization.canonical(for:)` maps every `kTCCService*` raw constant observed in real TCC.db files to the plan §4.1 short canonical name set (`fda`, `accessibility`, `automation`, `screen_recording`, `input_monitoring`, `microphone`, `camera`, etc.). Unknown constants pass through as `.other` with the raw value preserved on the artifact.

**Auth value / reason decoding.** `TCCAuthValue` and `TCCAuthReason` enum decoders for the SQLite integer columns. Defensive on unknown values — they map to `.unknown` rather than crashing the parser. The `isMDMGranted` / `isUserGranted` predicates feed into the risk scoring's mitigation path.

## Tests

1735 total across 322 suites passing. v1.13a-3-specific additions:

- `TCCRiskScoringTests` — 11 deterministic-score fixtures, one per scoring-table row + clamping + non-scored service.
- `TCCServiceNormalizationTests` — 7 raw-to-canonical + reverse mappings.
- `TCCAuthValue + TCCAuthReason decoding` — 5 boundary tests.
- `LiveDBSnapshotTests` — 4 cases (file produced, parsable as SQLite, dedup via sha, missing-source throws).

## What's intentionally NOT in this RC

- Codesign cross-reference for risk scoring: the `clientSignedByApple` / `clientHasKnownTeam` signals are wired through the scoring path but TCC-lite currently passes `false` for both. v1.13a-4 connects them via the codesign-resolve enricher when launchd-lite also needs the same hookup.
- `--diffAgainstBaseline` flag for `tcc.grant_added` / `tcc.grant_revoked` / `tcc.grant_changed` artifacts (plan §4.1 mentions the shape; the diff machinery is a follow-up).
- REG.db parsing — emits a coverage advisory note instead.
- `>=15 fixtures across macOS 12/13/14/15` per plan §4.1 CI gates: synthesized DBs cover the parser; the full fixture corpus lands ahead of v1.13a-5's release gate.
- MCP tool dispatch wiring — descriptors declared in the manifest; the actual `tcc_grants_for_service` / `tcc_automation_chain` MCP dispatch wires up with the meta-tool surface in v1.13a-1 follow-up.
