## MacCrab v1.4.9 — trusted browser helpers skip coordinated-attack

Same-day hotfix for v1.4.8. Google Chrome Helper legitimately
spans `credential_access` + `exfiltration` tactics during normal
Chrome sync (reading its own `Cookies` / `Login Data` DBs and
uploading to Google's cloud). NoiseFilter Gate 3 already
suppresses individual rule matches for browser helpers, but the
`CampaignDetector.checkCoordinatedAttack` path was still counting
tactics one tier up and emitting a HIGH "Coordinated Attack from
single process" campaign with zero visible contributing alerts.

### Fixed

- **`checkCoordinatedAttack` skips trusted browser helpers.**
  Parallels the existing `isAppleSystemDaemon` early-return:
  if the latest alert's process path matches
  `NoiseFilter.isTrustedBrowserHelper(path:)`, the coordinated-
  attack campaign path bails. Covers Google Chrome Helper,
  Safari Web Content, Firefox Content Process, Edge Helper,
  Arc Helper, and Electron app helpers.
