# MacCrab v1.21.3-rc.1

Forensics polish plus a store-plugin capability fix.

## Fixes

- **Built-in scanners appear immediately** in the Forensics view — previously
  they only showed up after you ran another scanner.
- **"Run on this Mac" no longer jars the window** — the plugin inspector now
  dismisses cleanly before the scan starts.
- Clearer wording: a refused or errored plugin names the real reason instead of
  internal jargon.

## Store / community plugins

- **A sandboxed community plugin can now read the files it declares.** It still
  runs fully contained (deny-default sandbox, no full-disk access), but its
  declared, consented read paths are served through a mediated broker — so a
  community forensic collector actually works on the store lane. Plugin authors
  use the new MacCrabPluginKit read API (documented in the Tier-B README).

## Notes

Release candidate for on-device validation; not published to the auto-update
channel. No detection-engine rule changes.
