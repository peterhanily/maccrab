## MacCrab v1.6.0 — Topology anomaly detection, battery-aware polling, analyst triage CLI

Minor release focused on genuine differentiation from commercial macOS EDRs.
Four headline additions:

### 1. TopologyAnomalyDetector — a new class of detection

No commercial macOS EDR catches "legitimate tools used in illegitimate process-tree
shapes" well. MacCrab now does. Four categorical invariants with high-weight
scoring:

- **launchd spawned shell** — `/sbin/launchd` directly spawning `/bin/sh`,
  `/bin/bash`, `/bin/zsh`, etc. Near-zero in normal operation; when it
  happens it's typically a persistence callback firing.
- **system process spawning staged binary** — a parent under `/System/`,
  `/usr/libexec/`, `/usr/sbin/`, or `/sbin/` spawning a child in `/tmp`,
  `/private/tmp`, `/var/tmp`, `/Users/Shared`, `~/Downloads`, or `.Trash`.
  Apple binaries have no business running user-staged content.
- **anomalous process fanout** — 20 + children from the same parent PID
  within 10 seconds. Catches fork storms, mass-scanning loops, and
  compromised servers.
- **deep process descent** — lineage depth > 15. Correlates with
  obfuscated shell chains (a weaker signal; adds to behavioral score
  rather than firing its own critical alert).

Emits through the existing BehaviorScoring pipeline so suppressions, alert
triage, campaign detection, and LLM investigation all work unchanged.

### 2. Battery and thermal state gating

Commercial EDR adoption on macOS laptops lives and dies by battery impact —
a tool that drains 30 % per day gets uninstalled regardless of detection
quality. MacCrab now scales down poll intervals on low-power mode and under
thermal pressure:

| Collector          | Base   | Low-Power Mode | Thermal Critical |
|--------------------|--------|----------------|------------------|
| ClipboardMonitor   | 2 s    | ~10 s          | ~7 s             |
| USBMonitor         | 10 s   | ~50 s          | ~35 s            |
| NetworkCollector   | 2 s    | ~6 s           | ~5 s             |
| EDRMonitor         | 120 s  | ~360 s         | ~300 s           |

Zero user configuration. Implemented via a single `PowerGate` helper in
`MacCrabCore/Utilities/`.

### 3. `maccrabctl why <alert_id>`

The single most common analyst question: "why did this rule fire on this
process?" The answer used to be "read the YAML rule and the event row and
figure it out yourself." Now:

```
$ maccrabctl why 7f3c2a9e-b84e-4e3d-9f2b-0c1a8b6d7e3f
────────────────────────────────────────────────────────────────────────
[HIGH] Network Extension Installed by Unsigned Process
   Rule id:    d1a2b3c4-0501-4000-a000-000000000501
   When:       2026-04-23 14:32:11
   Process:    stage1  (/tmp/stage1)
   MITRE:      attack.defense_evasion  attack.t1557
Predicates (6):
    1. file.path                         contains        /Library/Preferences/com.apple.networkextension.plist
    2. signer.type                       equals          apple
    3. process.image                     startswith      /System/, /usr/libexec/, /usr/sbin/
    ...
```

Handles both Sigma-rule alerts (dumps predicates + captured event fields)
and synthetic alerts from behavioral scoring / topology / campaign /
self-defense (explains the indicator family and points at the source file).

### 4. Three new detection rules

- **`persistence/system_launchdaemon_plist_replaced`** — writes to
  `/System/Library/LaunchDaemons/` by non-Apple-signed processes. Classic
  post-SIP-bypass rootkit persistence.
- **`defense_evasion/network_extension_unsigned_install`** — unsigned
  NEPacketTunnelProvider / NEDNSProxyProvider installs. Allowlists major
  legit VPN and EDR vendors.
- **`persistence/dock_persistence_entry_written`** — non-terminal process
  running `defaults write com.apple.dock persistent-apps`. Dock-injection
  persistence is subtle and under-monitored in other tools.

Total rule count: **420** (up from 417).

### Metrics export (bonus)

`/var/tmp/maccrab.metrics.json` now written every 30 s alongside the
heartbeat. Prometheus-textfile-style counters for external scraping:

