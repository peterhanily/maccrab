# v1.15.0-rc.1 — Composition-proof posture Analyzer

**Status: internal release candidate.** Not published to Sparkle appcast or Homebrew cask. Source tag only.

**v1.13a → v1.15 plan arc is feature-complete at this tag.**

## What landed

**`com.maccrab.forensics.posture-analyzer`** — first concrete Analyzer plugin and the composition-proof for the entire Mac Context Plugin Platform. Reads `tcc.grant` + `launchd.entry` artifacts from a case (collected by TCC-lite + launchd-lite, enriched by codesign-resolve) and emits five posture finding types:

| Finding | Severity bucket | What it catches |
|---|---|---|
| `posture.unsigned_persistence` | high (runs-as-root) / medium | launchd entry pointing to an unsigned binary that exists on disk — the post-compromise persistence surface |
| `posture.unfamiliar_team_persistence` | medium | launchd entry signed by a team that has only one entry across the inventory — unfamiliar third-party vendors persisting |
| `posture.automation_to_sensitive_target` | high | TCC Automation grant to Safari / Mail / Messages / Terminal / SystemEvents from a non-Apple client — the AI-agent reach surface |
| `posture.high_privilege_unsigned_combo` | critical | unsigned binary in launchd AND that same client has FDA / Accessibility / Screen Recording / Automation — the "both clauses required" finding from plan §7 v1.15 |
| `posture.permissioned_persistence` | medium | Developer-ID-signed app that persists AND holds broad TCC — the combination is the signal even when codesign is clean |

`posture.fingerprint_drift` and `posture.suspicious_correlation` are scoped out of this RC — `fingerprint_drift` is conditional on MCFP R2 (which selected static-only for v1.14; the baseline-corpus comparison waits for a future iteration), and `suspicious_correlation` is feature-flagged per plan §7 v1.15.

**`PluginRunner.runAnalyzer(id:, handle:, scope:)`.** Mirrors `runCollector` / `runEnricher`. Findings get committed back to the ArtifactStore as artifacts (`content_type = finding.findingType`), so they round-trip through `forensics.search_artifacts` / `forensics.posture_findings` MCP tools and through the future Dashboard Forensics tab via the same query path as any other artifact.

**`maccrabctl plugin run`** now dispatches by plugin type. Collectors get `runCollector`; Analyzers get `runAnalyzer`. Enrichers + Fingerprinters return a friendly "use 'maccrabctl fingerprint' or the forensics.* MCP tool surface" error.
