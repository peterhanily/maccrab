# MacCrab v1.18.0

A detection-quality release. A corpus-wide review of every detection — single-event rules, multi-step sequences, graph rules, and campaign logic — recovered detections that could never fire, removed the biggest false-positive sources, and fixed a filter bug that could miss real threats. It also adds an agent-session timeline and tightens privacy and reliability.

## Detection quality

- **Detections that could never fire now work.** A class of rules carried a condition the engine can never satisfy, so they silently never fired — including the Claude Code project-config hook-RCE detection (CVE-2025-59536 / CVE-2026-21852), several execution and credential-access rules, and multi-step sequence rules that were dropped at load time. They fire correctly now, and a build-time guard prevents a sequence rule from ever being silently dropped again.
- **Many fewer false positives.** AI-tool detections now require an actual AI-agent ancestor instead of firing on every developer who reads `~/.ssh` or `~/.aws`. TCC permission-grant detections resolve the requesting app's real code signature, so a legitimately-signed app installed outside `/Applications` is no longer flagged as if it were unsigned. And severities were recalibrated so a chatty rule can no longer escalate into an unsuppressible critical alert.
- **Command-and-control detections catch the common case.** A filter had been excluding shell-launched processes — exactly the lineage of a reverse shell or a `curl … | bash` payload — which suppressed the most common attack path. That's fixed; routine developer activity is handled by the noise filter instead.
- **Credential-read detections are active.** Reads of Safari, Notes, and password-manager databases and the system shadow-hash store are now observed, so the rules that watch for credential theft can actually fire.

## New

- **Agent session timeline.** A session-keyed view of an AI agent's activity — events, alerts, configuration changes, and tool calls on a single rail — with a cryptographically signed, verifiable evidence-bundle export and a session list in the dashboard.
- **Deprecated detections are labeled.** Rules that ship disabled now show a clear "Deprecated" badge in the Detection list and detail view.

## Dashboard, localization & accessibility

- **Localization now applies.** The bundled language packs load correctly across the dashboard — severity labels, navigation, and more render in the selected language, and Settings opens correctly in every language. Coverage is partial and expanding.
- **Density modes work.** The Basic / Standard / Advanced toggle (Settings → Appearance) tailors which workspaces appear in the sidebar; every detection stays active regardless.
- **Restore a suppressed campaign** from the dashboard (Alerts → Suppressed campaigns) instead of only the command line.
- **Reduce Motion** is honored across the workspace inspectors and the trace graph.
- **Command palette** "Create Detection Rule" now opens the rule editor.

## Fixed

- **Filter-negation correctness.** A multi-condition exclusion (`not (A and B)`) compiled to an over-broad form that could suppress real detections; it now negates correctly, closing a class of false negatives.
- **MCP-server privacy.** The MCP server no longer exposes the operator's username or home-directory paths to a connected agent.
- **No hung scorers.** Every Security Score subprocess is now bounded by a hard-deadline watchdog, so a stuck helper can't stall scoring.
- **CLI polish.** A rule-count off-by-one was corrected, CLI vocabulary was frozen, a `modules` command was added, and documented rule and test counts were unified to a single source of truth.
- **Storage diagnostics.** Storage-health diagnostics now report a rolling recent window instead of a cumulative since-boot total, so a long-past incident no longer shows indefinitely.

## Security

- **Custom rules install through the privileged inbox.** User-authored rules and rule overrides are written to a root-owned directory by the daemon — which the engine loads — instead of a group-writable directory it refused, so a rule saved from the dashboard actually takes effect.
- High-impact privileged-inbox actions now raise an observe-only self-protection alert.
- Exported session bundles are cryptographically signed and verified on import; path traversal in bundle handling is blocked.
- The release build aborts if a private key would ship in the bundle.

## Upgrade

- **Sparkle:** accept the in-app update prompt and relaunch.
- **Homebrew:** `brew upgrade --cask maccrab`
- **Manual:** download the DMG from the release page and replace MacCrab.app.
