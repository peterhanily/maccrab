## MacCrab v1.5.1 — False positive fixes (6 rules)

Patch release targeting false positives introduced or exposed since v1.5.0.

### Fixed

- **`ssh_agent_access_suspicious`: condition was OR, should be AND.**
  The rule fired on *any* file access by python3, curl, wget, or node — not
  just accesses to `SSH_AUTH_SOCK`. Changed `(selection or selection_process)`
  to `(selection and selection_process)` so the rule fires only when those
  processes specifically open the SSH agent socket. This was the root cause of
  spurious HIGH-severity alerts and cascading campaign noise for developer
  machines.

- **`csrutil_status_check`: removed `attack.defense_evasion` tag.**
  The rule carried both `attack.discovery` and `attack.defense_evasion`.
  A single csrutil execution therefore contributed two MITRE tactics, which
  was enough for the campaign detector to emit a "Coordinated Attack from
  single process" campaign. `csrutil status` is discovery (T1518.001), not
  evasion; evasion is handled separately by `sip_check_before_tampering`.

- **`mdm_enrollment_check`: added `filter_terminal`.**
  `profiles status` run from any shell or terminal app fired this rule.
  Added the standard terminal-parent exclusion list (consistent with all
  other discovery rules).

- **`hidden_file_created`: added `filter_known_apps` for Logitech Options+.**
  `logioptionsplus_agent` and related Logi daemon processes legitimately
  write dotfiles in the user home directory for config storage. Added an
  image-path filter covering `logioptionsplus`, `LogiMgr`, and `logi_`.

- **`process_listing_by_unsigned` / `system_enumeration_burst`: expanded
  terminal filter; added `filter_apple_child` to burst rule.**
  Added Cursor, VS Code (`Code`), node, and ruby to the parent-process
  exclusion list — these IDEs and runtimes commonly spawn `ps`, `top`, and
  system info commands as part of normal operation. The burst rule also gains
  `filter_apple_child: SignerType: apple` to suppress cases where the process
  itself is Apple-signed (e.g. `system_profiler`, `scutil`) but was launched
  by a non-terminal, non-system parent — a common pattern in software
  installers and update daemons.
