# v1.16.0-rc.3 — Safari-lite collector

**Status: internal release candidate.**

Plan §13.3 Safari-lite split. Inventories Safari's per-user state at metadata class — cookies / LocalStorage / IndexedDB (content class, plan §13.3 Safari-deep) stay in backlog.

## What landed

**`com.maccrab.forensics.safari-lite`** — Collector reading three Safari state files:

- `~/Library/Safari/History.db` (SQLite, snapshotted via `LiveDBSnapshot` since Safari may have it open). Emits `safari.history_visit` artifacts: url + domain + title + visit_count + load_successful + visit_id, ordered by observed time. Cap 5000 per run.
- `~/Library/Safari/Downloads.plist`. Emits `safari.download` artifacts: url + destination_path + identifier + mime_type + bytes_received / total_bytes + complete flag.
- `~/Library/Safari/Extensions/Extensions.plist`. Emits `safari.extension` artifacts: bundle_identifier + display_name + archive_file_name + enabled flag.

Privacy class `metadata` throughout. Sigma rules + Analyzer findings can match on `safari.history_visit.domain` for known-bad-domain checks; on `safari.download.url` for malware-download correlation; on `safari.extension.bundle_identifier` for known-abuse extensions.

## What's not in this RC

- **Reading List + Bookmarks** — `Bookmarks.plist` has a more complex tree structure; parser deferred. The plan §13.3 lite scope still mentions them; they land in a follow-up when the dashboard's bookmark drill-down is built.
- **CloudTabs** — `CloudTabs.db` parsing deferred; lower signal than History.
- **Safari-deep** (cookies / LocalStorage / IndexedDB / WebSQL) — content-class, plan §13.3 explicit split.
