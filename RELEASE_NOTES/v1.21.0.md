# MacCrab v1.21.0

A features-and-correctness release.

## Dashboard

- **Gap-free customizable Overview.** The grid now uses a masonry layout, so mixed-height widgets pack together cleanly at any window size — no more ragged gaps when you resize or rearrange.
- **New widgets:**
  - **Protection Coverage** — live sensor health ("N of M sensors healthy") so a silently-stopped collector is obvious.
  - **Top Firing Rules** — your noisiest detections by fire count, with last-fired time and real rule names.
  - **Crabby** — a Tamagotchi-style companion built from the MacCrab pixel crab. He reacts to your live security state (weather, mood, a coverage shield, an AI sidekick robot, a mood beacon) and you can Pet, Feed, Acknowledge an alarm, or jump to Investigate.
- **Catalog plugin management.** An installed plugin now shows a working **Update / Reinstall / Remove** with an "up to date" status, instead of offering an install that errored.
- **Reset to default** confirms before discarding a customized layout.
- **No more false "Protection degraded".** The banner could latch a "no rules loaded" reading after an update; it now re-checks until rules load and reflects real engine health. A low posture score (e.g. the firewall is off) is shown by the Security Grade tile, not the degraded banner.

## Detection & scoring

- A broad "trust this process/host" allowlist can no longer silence an active command-and-control or credential-theft alert — only an explicit per-rule entry can.
- More honest security score: XProtect (Apple-managed) is no longer false-flagged as outdated, and the Screen Lock factor no longer false-warns on modern macOS.

## AI agent integration (MCP)

- Forensic plugin platform parity between the CLI and the MCP server: list cases, search the signed catalog, run all scanners on a case, and — with the human-granted **response** capability — install, update, uninstall, or pin plugins. A read-only `get_daemon_config` companion was also added. Every change runs through the same verified, audit-logged path the CLI uses and is off by default.

## Reliability & accessibility

- Oversized event payloads are always stored as valid, queryable records.
- Hardened the live rule reload (SIGHUP).
- Accessibility (VoiceOver) and localization polish across the new widgets and the install-consent sheet.
