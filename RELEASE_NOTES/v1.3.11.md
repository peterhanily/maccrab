## MacCrab v1.3.11 — Quieter alerts, cleaner dashboard

This release is driven by real field data. One workstation was showing
**1,144 cross-process alerts** and dozens of false positives on Wi-Fi,
EDR, and Gatekeeper rules over 24 hours. Three fixes resolve the vast
majority of that noise, plus two dashboard bugs.

---

### Fewer false positives

- **Rules are more precise.** A Sigma compiler bug was collapsing
  multi-selection rules into flat predicate lists, so rules like
  *Wi-Fi Attack Tool* fired on any commandline containing `-s` and
  *EDR Remote Session* fired on any commandline containing `connect`.
  Fixed compiler-side; all 380 rules recompiled with correct
  AND/OR semantics.
- **Vendor worker chains suppressed.** Cross-process correlator now
  ignores chains where every event comes from the same executable,
  app bundle, tool-version directory, or process name — the pattern
  produced by logger fan-outs like GoogleUpdater, not attacker
  convergence.
- **Broken unicode rule retired.** *Invisible Unicode in Source Code*
  was matching on filename extension alone with no content check,
  firing on every JSON/YAML/py write. Disabled until content-based
  regex is supported.

### Dashboard

- **Bulk-dismiss for campaigns.** New Select mode in the Campaigns
  view: tick multiple campaigns, click *Dismiss Selected*.
  Delete-key dismisses, Escape cancels.
- **Sidebar Alerts count no longer double-counts campaigns.**
  Campaign-prefixed ruleIds now show only in the Campaigns badge,
  not in Alerts.

### Docs

- `UPGRADE.md` documents the manual DMG drag-n-drop upgrade path.
  Dragging a new DMG onto an existing install triggers the
  SystemExtension `.replace` flow automatically on next launch.

---

### After updating

If you had noisy alerts on the previous version, they'll stop
appearing in new events. Existing alerts in the database stay until
you dismiss or clear them — the Campaigns *Select → Dismiss
Selected* flow is the fastest way to clear a batch.

Spot-check the noise drop one hour after update:

```
sudo sqlite3 "/Library/Application Support/MacCrab/events.db" \
  "SELECT rule_id, COUNT(*) FROM alerts
   WHERE timestamp > strftime('%s','now','-1 hour')
   GROUP BY rule_id ORDER BY 2 DESC;"
```

**Compatibility:** backwards-compatible. No schema migration, no
re-approval of the system extension, no data loss. 592 tests pass.
