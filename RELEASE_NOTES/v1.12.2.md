# MacCrab v1.12.2 — Sparkle install fix

> [!IMPORTANT]
> **If you are currently on v1.12.0 or v1.12.1**, your installed app
> carries broken Sparkle helpers and **cannot auto-update to v1.12.2**.
> Sparkle will surface the same "An error occurred while running the
> updater" message. You must upgrade manually:
>
> **Option A — Homebrew (recommended)**
>
> ```
> brew update
> brew upgrade --cask maccrab
> ```
>
> **Option B — Manual DMG**
>
> 1. Download
>    [MacCrab-v1.12.2.dmg](https://github.com/peterhanily/maccrab/releases/download/v1.12.2/MacCrab-v1.12.2.dmg)
> 2. Quit MacCrab (menubar → Quit, or right-click → Quit in Activity Monitor)
> 3. Open the DMG, drag MacCrab.app to /Applications (replace the
>    existing copy)
> 4. Launch the new MacCrab.app. Approve the SystemExtension prompt
>    if it appears (it usually doesn't on in-place reinstalls).
>
> Once you are on v1.12.2, future Sparkle in-place upgrades will
> work normally — this is a one-time manual step.
>
> **If you are on v1.11.x or earlier**, regular Sparkle upgrade works
> — open MacCrab.app and choose "Check for Updates…" from the menu.

---

v1.12.2 is the same code as v1.12.1 plus a single build-pipeline fix
for the broken Sparkle auto-updater that surfaced as "An error
occurred while running the updater" on every v1.12.1 upgrade attempt.

## What was broken

The v1.12.0 RC28 audit fix added `--deep --entitlements <APP_ENT>
--force` to the outer `.app` codesign pass to seal
`Resources/compiled_rules/` into the bundle signature. `--deep
--force` re-signs every nested Mach-O, and `--entitlements`
propagates MacCrab's main-app entitlements to all of them. Sparkle's
nested helpers — `Autoupdate`, `Updater.app`, `Downloader.xpc`, and
critically `Installer.xpc` — all ended up with
`com.apple.developer.system-extension.install` and
`keychain-access-groups = [79S425CW99.com.maccrab.shared]` baked
in. macOS refuses to launch a Sparkle XPC helper with that
entitlement combo, which surfaces as the generic "An error occurred
while running the updater".

## What changed

Single line in `scripts/build-release.sh`: the outer
`codesign --deep --entitlements …` pass now also passes
`--preserve-metadata=entitlements`. Net effect:

- Resources/* (compiled_rules, rules, Compiler) still get sealed
  into the outer signature (the RC28 goal).
- Nested Mach-Os keep the entitlements they were signed with at
  their own (earlier) step — empty/Sparkle-default for Sparkle's
  helpers, `APP_ENT` for the inner MacCrab executable.

Verified post-build: `codesign -d --entitlements -
Sparkle.framework/Versions/B/XPCServices/Installer.xpc` returns
empty entitlements (correct); outer MacCrab executable still
carries `com.apple.developer.system-extension.install` (correct).

## How to get v1.12.2

**If you're already on v1.12.0 or v1.12.1**: your installed app
carries the broken nested entitlements on its OWN Sparkle helpers,
so its Sparkle can't install v1.12.2 either. Upgrade via:

```
brew upgrade --cask maccrab
```

or download the v1.12.2 DMG from:

https://github.com/peterhanily/maccrab/releases/tag/v1.12.2

Once you're on v1.12.2, future Sparkle upgrades will work normally.

**If you're on v1.11.x or earlier**: your Sparkle helpers are still
correctly signed, but they still try to launch v1.12.2's helpers
during install — fixed in this release. Sparkle upgrade to v1.12.2
should work directly.

## No Swift source changes

Every Swift file is identical to v1.12.1. The only diff vs v1.12.1
is the one-line build-script change and version-marker bumps.
