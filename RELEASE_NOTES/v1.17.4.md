# MacCrab v1.17.4 — reliability + detection-quality release

This release bounds runaway disk and CPU growth, cuts alert noise on
everyday developer and AI-tool activity, revives several detection paths
that had silently stopped firing, and makes the engine-side AI backend
actually reachable.

## Fixed

- **Unbounded disk growth.** The causal-graph store could grow without
  limit (observed at 17 GB) because its entity/edge data was never
  pruned. It now has time-based retention, a configurable size cap, and
  an orphan sweep.
- **Sustained CPU.** The engine no longer feeds every event into the
  causal graph — it applies the same noise filter as the event store and
  writes each event's graph data in a single batched transaction.
- **False "coordinated attack" campaigns on legitimate tooling.** Deep
  shell nesting and keychain/`sudo` rules no longer flood alerts on
  routine developer / AI-agent activity (full process-lineage filtering,
  severity recalibration, and an AI-attribution campaign guard).
- **Credential-file READ detection.** Rules that flag reads of SSH keys,
  cloud credentials, and crypto-wallet data can now fire — the engine
  observes file *opens* for those specific paths (previously only writes
  were seen, so the read rules were effectively dead).
- **Content-scanning rules revived.** A collector/enricher mismatch had
  silently disabled the `FileContent` rules (AI-skill / MCP / workflow
  poisoning detections); fixed and covered by tests.
- **Dead-category rules.** Six rules that could never fire are now marked
  deprecated, and a compiler check prevents this class of dead rule from
  shipping again.
- **MCP server** now speaks spec-conformant newline-delimited JSON and no
  longer drops or corrupts back-to-back messages.

## Added / Improved

- **Configurable storage caps** for the causal graph, traces, reports,
  and auto-generated rules (`daemon_config.json`); the events
  write-ahead log is now size-bounded.
- **Engine-side AI backend now works** — the dashboard's AI Backend
  settings reach the detection engine, the engine verifies the configured
  local model is installed before enabling LLM features, and it rejects
  non-loopback endpoints by default.
- **Forensic-scan retention** is now enforced automatically.

## Changed — please read before upgrading

- **Forensic scans now auto-delete after 1 year by default** (previously
  kept forever). To keep everything, set Settings → Scan retention →
  *Never*.
- **The default `alerts.jsonl` tail is no longer written.** Alerts remain
  in the queryable alert store; add a `file` output in
  `daemon_config.json` for a local NDJSON copy.
- **Credential-path file-open monitoring is on by default.** Set
  `subscribe_file_open_events: false` to disable it.

## Upgrade

- **Homebrew:** `brew upgrade --cask maccrab`
- **Sparkle:** accept the in-app update prompt and relaunch when asked.
- **Manual:** download the DMG from the release page and replace MacCrab.app.
