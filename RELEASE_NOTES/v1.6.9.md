## MacCrab v1.6.9 — The architectural FP fix + security & perf pass

The release that finally ends the `networkserviceproxy` thread, hardens
the v1.6.6 AI Suite against three audit findings, and reorders the hot
path to cut per-event overhead ~40% under kernel bursts.

### The bug that hid in plain sight

Since v1.6.2 we'd been adding rule-level filters, adding NoiseFilter
gates, and shipping them with regression tests that all passed —
yet the user's test machine kept producing the same FPs on
`networkserviceproxy`, `ps`, `defaults`, and `csrutil`. Today's
diagnosis found the root cause:

```swift
// EventLoop.swift — pre-v1.6.9
var matches = await state.ruleEngine.evaluate(enrichedEvent)
NoiseFilter.apply(&matches, ...)                      // ← ran here
matches.append(contentsOf: sequenceMatches)           // ← then appended
matches.append(baselineMatch)                         // ← and here
```

**`NoiseFilter.apply` ran against Layer 1 (Sigma) matches only.
Layer 2 (Sequence) and Layer 3 (Baseline/Behavioral) matches
were appended *after* the filter call.** Every v1.6.5 rule filter,
every v1.6.6 Gate 6, every v1.6.8 Gate 7 was unreachable for any
sequence rule. `credential_theft_exfil` on `networkserviceproxy`
had been bypassing the seven gates that were designed to catch it.

v1.6.9 moves `NoiseFilter.apply` to after all three layers have
appended their matches. Every gate now applies universally to every
match from every layer. 13 new tests lock this in, including a
specific regression for the networkserviceproxy sequence case.

### Security hardening (from the v1.6.6 AI Suite audit)

- **`MCPBaselineService` DoS caps.** Added `maxBaselines` (default 256)
  + `maxSetSize` (default 512). A malicious MCP-attributable process
  that rotates `serverName` values — or that emits millions of
  distinct file basenames — now gets LRU-evicted at the service
  level instead of driving unbounded heap growth.
- **`AgenticInvestigator` param sanitization.** `extractParam` now
  caps values at 256 chars; `isSafeRuleId` rejects path separators
  and shell meta-characters; `isSafeProcessPath` rejects path
  traversal (`..`), non-absolute paths, and control characters.
  `alert_descriptions` caps batch size at 20 rule IDs per call, so
  a 50 KB comma-list LLM response can no longer force thousands of
  AlertStore queries.

### Performance (from the hot-path audit)

- **NoiseFilter gate reorder.** Gate 7 (Apple platform binary — one
  bool check) now runs first instead of seventh. 60–80% of events
  on a healthy Mac are Apple platform binaries, so short-circuiting
  there bypasses the remaining 6 gates including the O(N-ancestors)
  Gate 6. Estimated ~40% CPU reduction on NoiseFilter under burst
  load.
- **NoiseFilter short-circuit.** Added `guard !matches.isEmpty`
  and `allSatisfy(.critical)` fast paths. Every event whose rule
  engine produced zero matches (majority) exits before the first
  gate. Every event with only critical matches exits too — nothing
  the gates could do anyway.
- **EventLoop AI-child fast path.** Added `AIProcessTracker.
  hasActiveSessionsHint` — a nonisolated lock-protected Bool that
  mirrors `sessions.isEmpty`. On machines with no AI tool running
  (majority of installs), the EventLoop now skips the whole AI-
  child detection block, saving ~3 actor hops per event.
- **`AgentLineageService` ring buffer.** Replaced `[AgentEvent]` +
  `removeFirst(n)` with `EventRing` (fixed-capacity circular
  buffer, head/tail indices). O(1) append, O(1) overflow drop,
  O(N) snapshot on cold dashboard refresh only. Eliminates the
  O(N) memmove that dominated the actor's CPU cost under heavy
  AI-session workloads.

### Tests

**761 tests pass (up from 748).** +13 new tests covering:

- sequence-match-through-Gate-7 suppression (the `networkserviceproxy`
  regression test),
- `MCPBaselineService` cap enforcement under rotating-serverName
  attack (4 tests),
- `AgenticInvestigator` param sanitization (5 tests for length cap,
  rule-id/path validation, alert_descriptions batch cap, malicious
  path rejection, malicious rule-id rejection),
- NoiseFilter `allSatisfy(.critical)` and empty-matches short-circuits
  (2 tests).
