# MacCrab v1.21.2

Enables first-party forensic plugin execution.

## Fixes

- **First-party forensic plugins run again.** A signed first-party plugin (e.g.
  "Mac Security Posture Pro") ran but committed no results, because the
  first-party execution anchor was never configured — so it fell through to the
  sandboxed lane and produced nothing. The anchor is now configured, so a
  first-party plugin runs on its intended (unsandboxed, Full-Disk-Access) lane
  and collects its full artifact set.
- **Legible skip reasons.** When a plugin is refused or errors during a
  Forensics scan, the done banner now shows the real reason instead of a generic
  "not a built-in or installed forensic plugin".

## Notes

No detection-engine rule changes (436 single + 41 sequence + 6 graph). Signed,
notarized, and delivered via Sparkle auto-update. Third-party / sideloaded
plugins are unaffected — they still run sandboxed, or are refused fail-closed.
