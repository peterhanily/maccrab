# v1.16.0-rc.1 — AppleScript runtime monitor

**Status: internal release candidate.** Not published to Sparkle appcast or Homebrew cask.

First §9.2 backlog item to ship — the explicit v1.16 candidate from plan §13.5. Catches the 2025-2026 fileless macOS infostealer class (MacSync / DigitStealer / AMOS variants) that runs entirely inside Apple-signed binaries (`osascript` / `osacompile` / JXA) where the payload arrives via argv, not a file at rest.

## What landed

**`com.maccrab.forensics.applescript-runtime`** — Collector that queries `MacCrabCore.EventStore` (populated by the Track 1 ES collector via `NOTIFY_EXEC`) for exec events whose `process.executable` matches an AppleScript-runtime path:

- `/usr/bin/osascript`
- `/usr/bin/osacompile`
- `/usr/bin/osalang`
- any `*/JavaScriptCore.framework/.../Resources/jsc` (jsc lives under a versioned framework path)

For each match emits an `applescript.invocation` artifact:
- `runtime` — exact executable path observed.
- `pid`, `ppid`, `command_line`, `arguments_json` (full argv preserved).
- `user_name` — owning user account.
- `parent_pid`, `parent_executable` (when ancestry available).
- `decoded_payload_*` — heuristic base64-decoded payload when any argv element looks base64-shaped (≥40 chars, base64 alphabet, decodes to ≥85% printable text). The argv itself is preserved alongside.
- `exec_observed_at_iso` — NOTIFY_EXEC timestamp.

**Privacy class: `content`.** Per plan §13.5's inline §10.7 justification:

1. The argv IS the malware. Capturing argv with default-off would defeat the plugin's purpose.
2. The capture window is at-exec only — does not read AppleScript files at rest.
3. The privacy class is still labelled content; dashboard shows the content-class warning chip; MCP exposure gated by `case.ai_content_allowed` per §10.8.

Pass 2026-D enforces at INSERT: plaintext cases reject every `applescript.invocation` artifact; the plugin's `CollectionResult.notes` reports the rejection and points the operator at "create an encrypted case to capture these."

## What this catches that v1.13a / v1.15 missed

The TCC + launchd substrate catalogs persistence + permission surfaces — not exec events. The AppleScript runtime monitor is the first Collector to consume Track 1's exec stream and surface argv-resident attacks. Concrete example: a `osascript -e "do shell script \"curl ... | sh\""` invocation produces an `applescript.invocation` artifact with the full -e string; the dashboard / MCP query "applescript_with_base64_payload" surfaces the decoded payload when it's base64-encoded.

## Tests

1776 across 329 suites, all passing. v1.16.0-rc.1 additions:

- `AppleScriptRuntimePlugin: runtime-binary matcher` — 5 cases (`/usr/bin/osascript` + `/usr/bin/osacompile` matches, `jsc` framework-path suffix match, random binaries don't match, lookalike paths anchored correctly).
- `AppleScriptRuntimePlugin: base64 heuristic` — 5 cases (recognizes encoded shell, ignores short args, ignores non-base64 long strings, filters binary-decoded low-printability content, picks the first base64-shaped argv element in a multi-arg list).
- `AppleScriptRuntimePlugin manifest` — 3 cases (Pass 2026-A validate, output declares `.content` privacy class, namespaced id).
