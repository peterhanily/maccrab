# MacCrab v1.21.0-rc.1

A fixes-and-polish release candidate, with the forensic plugin platform now manageable from the MCP server.

## Dashboard

- **Overview customization fills the grid neatly.** Resizing a panel no longer leaves an odd gap at the end of a row — rows now justify to the full width.
- **Forensics catalog handles installed plugins properly.** An already-installed plugin now shows "Installed · up to date" (or "update available"), with an **Update** action only when a newer version exists and a **Remove** action to uninstall — instead of offering a re-install that failed with a "destination already exists" error.
- **Clearer trust gates.** The third-party plugin install consent is shown as a prominent warning, and granting an AI agent defense-affecting settings now requires an explicit confirmation.

## Accessibility

- VoiceOver can now reorder, resize, and hide Overview panels while customizing (previously mouse-only), and the panel drag animation honors **Reduce Motion**.

## Detection & reliability

- A broad "trust this process/host" allowlist can no longer silence an active command-and-control or credential-theft alert — only an explicit per-rule entry can.
- Oversized event payloads are stored as valid, readable records (a rare truncation edge previously made a few events un-queryable).
- Hardened the live rule reload triggered by SIGHUP.

## AI agent integration (MCP)

- The forensic plugin platform now has CLI/MCP parity: an AI agent can list cases, search the signed catalog, run all scanners on a case, and (with the human-granted **response** capability) install, update, uninstall, or pin plugins. A read-only `get_daemon_config` companion to `set_daemon_config` was also added. All plugin changes go through the same verified, audit-logged path the CLI uses, and remain off by default.
