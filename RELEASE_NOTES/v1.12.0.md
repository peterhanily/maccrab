# MacCrab v1.12.0 — Supply chain detection + intent posterior

v1.12.0 broadens detection coverage across the supply-chain attack
surface (npm / PyPI / Homebrew) and adds two pieces of analyst-facing
machinery: a Bayesian posterior over attacker goals per process tree,
and a graph rule evaluator that fires on multi-entity patterns inside
materialized traces.

The release is **detection-only**. Every new rule emits an alert; no
auto-response is wired in. Honey-prompts and honeyfile bait remain
opt-in behind `MACCRAB_DECEPTION=1`.

Sparkle in-place upgrade from v1.11.1 lands the changes without
manual action other than relaunching MacCrab.app when prompted.

## What's new

- **Supply-chain rule wave.** Coverage for the September 2025
  Shai-Hulud worm class plus the April–May 2026 Mini Shai-Hulud /
  Lightning PyPI / TanStack CVE-2026-45321 incidents. New rules
  target the worm-shape signals: credential-file read followed by
  npm-publish-to-self, GitHub /user/repos POST from non-`git` parent,
  PyPI twine upload from a non-interactive lineage, registry OIDC
  token exchange from a non-interactive parent.

- **Typosquat scoring.** `TyposquatDatabase` ships a Damerau-
  Levenshtein scorer with Unicode confusable folding. The
  bundled top-package JSON corpora (~200 npm + ~200 PyPI entries)
  load from MacCrabCore's resource bundle and can be replaced
  post-install with a larger list (operator's choice; MacCrab does
  not auto-fetch). PackageScanner eagerly scores every installed
  package; results surface in the dashboard's Intelligence →
  Package Freshness inspector.

- **Bayesian intent posterior.** `BayesianIntentEngine` accumulates
  evidence per process tree as kernel events flow through the daemon.
  An alert fires when the top non-benign goal probability crosses
  0.85 AND ≥3 distinct evidence types accumulate (the thresholds are
  intentionally strict — single-event signals already alert via the
  Sigma rule pipeline).

- **Graph rule evaluator.** Six multi-entity rules in
  `Rules/graph/*.json` (including the headline
  `maccrab_worm_self_propagation` pattern) now evaluate on every
  materialized Trace from the TraceGraph pipeline. Matches route into
  the standard alert sink.

- **AI-agent context bait.** `HoneyPromptManager` deploys
  decoy `CLAUDE.md.canary`, `SKILL.md`, and `cursorrules.canary`
  files under `~/Library/Application Support/MacCrab/decoys/`. An
  AI agent that walks its own context window and pulls a decoy
  generates a tagged file-read event.

- **MCP server expansion.** Eight new tools added for AI-agent
  integration: `check_typosquat_score`, `scan_package_content`,
  `analyze_package_metadata`, `verify_package_attestation`,
  `classify_package_intent`, `predict_next_technique`,
  `score_text_style`, `get_intent_posterior`.

## Footnotes on scope

- **Graph rule corpus is small.** Six rules ship in v1.12.0. The
  evaluator is the substrate; the rule library will grow over the
  v1.12.x line.

- **Heavier supply-chain analyzers (registry metadata, Sigstore /
  PEP 740 attestation, content scan) are reachable from MCP today.**
  Eager dashboard enrichment is gated to avoid hammering registry
  HTTP on every panel refresh.

- **Credential-read coverage is shell-invoked only.** The Bayesian
  intent posterior + IntentClassifier verdict pipeline registers a
  `credentialRead` evidence event when a credential-shape path
  (`~/.aws/credentials`, `~/.ssh/id_*`, `.npmrc`, `.pypirc`, etc.) is
  read via a shell-invoked reader: `cat`, `less`, `head`, `tail`,
  `grep`, `awk`, `jq`, `security`, `openssl`, `gpg`, `base64`,
  `xxd`, `od`, `strings`. Reads via library calls inside a process
  (`fs.readFileSync` from Node, `open()` from Python, etc.) are not
  captured because ESCollector does not subscribe to
  `NOTIFY_OPEN`/`NOTIFY_READ` in v1.12.0. Worms that read creds via
  a libc path are still caught by the per-event Sigma rules layer
  (`credential_access/*.yml`) and the graph-rule layer's
  `credential_file` entity type, just not by the intent posterior.
  Full ES OPEN subscription is a v1.13.x line item.

- **Honey-prompts + honeyfiles are opt-in (`MACCRAB_DECEPTION=1`).**
  Operators who want canary deployment must set the env var on the
  daemon and run `maccrabctl deception deploy`. A dashboard one-time
  prompt to enable deception is queued for v1.12.1.

- **Dev-mode gating.** `MACCRAB_DEV_MODE=1` halves AI-tool indicator
  weights in `BehaviorScoring` and raises the kill-chain tactic
  threshold from 4 to 5. Use during MacCrab development to suppress
  self-firing alerts; do not set in production deployments.

## Stats

- 463 YAML rules total under `Rules/` — 424 single-event + 39 sequence
- 6 graph rules under `Rules/graph/`
- 1490 unit tests across 284 suites — all passing
- Minimum macOS 13 (Ventura), same as v1.11.x

## Upgrade notes

- No schema migrations. EventStore stays at v5; alerts.db, tracegraph.db
  schemas unchanged from v1.11.x.
- No new mandatory daemon_config keys. Defaults are baked in for the
  new thresholds (intent posterior 0.85, ≥3 evidence types).
- The `~/Library/Application Support/MacCrab/decoys/` directory is
  created only when `MACCRAB_DECEPTION=1` is set in the daemon env.
