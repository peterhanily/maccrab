# MacCrab v1.19.3

A plugin-lifecycle and signal-quality release: installed forensic plugins can now
be updated and pinned from the signed catalog, routine developer-tooling activity
no longer raises high-severity noise, and installed plugins are managed in one place.

## Plugin updates

- **Update installed plugins from the catalog.** `maccrabctl plugin update <id>`
  fetches the catalog's current version, compares it to what's installed, and
  re-installs through the full signature-verified path. Patch updates apply
  directly; minor and major updates require confirmation.
- **Update indicator in the dashboard.** Installed plugins in *Run a scan* now show
  an "Update available" badge and an Update button when a newer version is published.
- **`maccrabctl plugin check-updates`** reports which installed plugins have a newer
  catalog version (`--json` for tooling).
- **Pinning.** `maccrabctl plugin pin <id>` freezes a plugin at its installed
  version and skips update offers; `--unpin` restores updates. A pin never blocks
  a revocation — a pinned plugin that is later revoked is still quarantined.
- **Forward-only.** Updates and re-installs are forward-only; a downgrade to an
  older version is refused.

## Quieter alerts

- **Routine developer-tooling activity is down-weighted, not hidden.** Bundlers,
  package-manager runtimes, language toolchains, and AI coding tools generate a
  lot of benign multi-stage behavior on a developer's machine. Those alerts now
  surface at low severity for review instead of raising high/critical noise.
- **High-signal detections are unchanged.** Credential and keychain access,
  ransomware and data-destruction, security-tool tampering, persistence, and
  AI-tool credential exfiltration still escalate at full severity, including on
  developer paths — a malicious package is a primary delivery vector for exactly
  those, so they are never quieted.
- MacCrab's own background processes no longer generate alerts about themselves.

## Forensics

- **Unified plugin management.** Installed plugins are now managed directly in
  *Run a scan* — run, verify, update, or uninstall from one place. The separate
  "My Plugins" tab has been removed.
- Fixed a case where an installed store plugin had no Run control.

## Compatibility

No configuration changes are required. macOS 13 (Ventura) or newer.
