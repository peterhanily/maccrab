# MacCrab v1.17.5 — security & detection-precision release

A focused hardening release on top of v1.17.4. It closes a local
exfiltration gap in the engine's LLM-configuration path, completes
credential-read detection coverage, sharpens several rules against
evasion and false positives, and fixes a Homebrew upgrade that could
disturb the system extension.

## Fixed

- **LLM endpoint hardening.** The detection engine now strictly validates
  that a configured local LLM endpoint is genuinely loopback before
  trusting it. A hostname crafted to *look* local can no longer redirect
  the engine's LLM traffic to an external host or bypass prompt
  sanitization.
- **Homebrew upgrades no longer disturb the system extension.** A
  `brew upgrade` previously ran the uninstall steps and could tear down
  the Endpoint Security extension mid-upgrade (dropping protection and
  prompting for approval). The extension now stays active across upgrades;
  removing it on a true uninstall is handled by the in-app control or the
  bundled uninstall script.
- **Cryptocurrency-wallet read coverage.** Credential-read detection now
  covers the full set of wallets the rule targets — MetaMask, Phantom,
  Atomic, Coinomi, and others — instead of a subset.
- **Credential reads in the trace graph.** File-open/read events now enter
  the causal graph, so multi-step credential-access traces (e.g. "read SSH
  key, then connect out") materialize correctly.

## Improved

- **Detection precision.** Shell-nesting and keychain/`sudo` filters now
  match process lineage at path-component boundaries, closing an evasion
  gap (such as staging a payload under a `node_modules` path) and removing
  collisions with unrelated processes. The Wi-Fi password-extraction rule
  again matches the common `security find-generic-password -ga` form.
  Keychain open-monitoring no longer admits routine system keychain
  access.
- **MCP error reporting.** Tool failures and invalid arguments now report
  consistently as errors, so AI agents don't mistake them for results.
- **Dashboard.** The Investigation view reads engine LLM health off the
  main thread.

## Upgrade

- **Homebrew:** `brew upgrade --cask maccrab`
- **Sparkle:** accept the in-app update prompt and relaunch when asked.
- **Manual:** download the DMG from the release page and replace MacCrab.app.
