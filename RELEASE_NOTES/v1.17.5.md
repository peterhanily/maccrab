# MacCrab v1.17.5

A feature and hardening release.

## New

- **ClickFix detection.** Flags when a shell command pasted from the clipboard (e.g. `curl … | bash`) is then executed — the dominant 2026 macOS infostealer delivery technique, which sidesteps Gatekeeper because nothing is downloaded and launched.
- **Sort and filter columns.** Click any table header to sort, and filter rows, across Alerts, Detection rules, Intelligence, and the other tables.
- **Tune built-in detections.** The built-in detections now appear in the Rules list and can be muted or have their severity overridden from the dashboard.
- **Surrounding events survive pruning.** An alert's detail now shows the events captured when it fired, even after the originating events have aged out of the live store.

## Fixed

- **LLM endpoint hardening.** The engine now strictly validates that a configured local LLM endpoint is genuinely loopback before trusting it, so a hostname crafted to *look* local can't redirect LLM traffic to an external host or bypass prompt sanitization.
- **Credential-read coverage.** A set of credential and cryptocurrency-wallet read detections that previously could not fire now do, and credential file reads enter the causal trace graph so multi-step credential-access chains materialize.
- **Homebrew upgrades no longer disturb the system extension.**

## Improved

- Sharper detection tuning across the rule set — fewer false positives and tighter process-lineage matching against evasion.
- MCP tool failures and invalid arguments now report consistently as errors.
- Numerous stability and storage-retention fixes.

## Upgrade

- **Sparkle:** accept the in-app update prompt and relaunch.
- **Homebrew:** `brew upgrade --cask maccrab`
- **Manual:** download the DMG from the release page and replace MacCrab.app.
