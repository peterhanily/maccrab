# MacCrab v1.19.2

A plugin-store and forensics-fidelity release: the signed plugin store is now
installable end to end, evidence is exportable as a portable bundle, and several
built-in forensic collectors are corrected.

## Plugin store

- **Installing signed first-party plugins now works end to end.** Browse the
  catalog, click Install, and the plugin is downloaded, signature-verified, and
  trusted in one step — both from the dashboard and from the CLI
  (`maccrabctl plugin install <id>`).
- **`maccrabctl plugin search`** lists the signed catalog from rave.maccrab.com.

## Forensics

- **Evidence export.** `maccrabctl scan export <scan-id>` and
  `maccrabctl evidence export --scan <scan-id>` write a `.maccrabevidence`
  bundle — every artifact in the case plus an integrity hash, verifiable offline.
- **Built-in collector fixes:**
  - **FSEvents** now reads the modern Data-volume store
    (`/System/Volumes/Data/.fseventsd`) and mounted volumes — it previously
    looked at a root path that no longer exists on current macOS, so it returned
    nothing.
  - The **plist**, **DMG/PKG**, and **Office document** file-analyzers honor an
    operator-supplied path instead of only scanning a default location.
  - **Office documents** now extract their created/modified timestamps.
  - **Biome** enumerates the current stream layout on recent macOS.
- Forensic artifact tables now render every field correctly in the dashboard.

## Compatibility

- All trust, signing, privacy, and detection behavior is unchanged from v1.19.1.
- Requires macOS 13 or later.
