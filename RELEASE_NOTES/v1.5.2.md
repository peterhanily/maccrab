## MacCrab v1.5.2 — Dashboard UX fixes

Patch release with two dashboard fixes. No new detection rules, no schema changes.

### Fixed

- **Full Disk Access banner now clears correctly after granting both entries.**
  The previous sysext FDA detection used a WAL file mtime heuristic — it
  checked whether the daemon had written to `events.db` in the last 5 minutes
  rather than checking TCC state directly. This meant the "FDA required" banner
  would stay up after granting access if the daemon was in a quiet period, and
  it would disappear before access was actually granted if the daemon happened
  to be active for other reasons.

  MacCrab now queries `~/Library/Application Support/com.apple.TCC/TCC.db`
  directly for the sysext's authorization row whenever the app itself has FDA.
  The result is authoritative and updates within one 10-second dashboard poll
  cycle. The WAL heuristic is kept only as a fallback for the case where the
  app doesn't yet have FDA (and therefore can't read TCC.db).

- **FDA banner is now a two-row checklist.**
  The banner previously showed a paragraph of text that changed based on
  which principal was missing. It was easy to miss that both "MacCrab" and
  "MacCrab Endpoint Security Extension" need separate FDA grants. The banner
  now shows both items as a checklist with ○/✓ indicators so the pending
  step is visible even after completing the first grant.

  The "Reveal MacCrab in Finder" shortcut is now conditional: it only appears
  when the MacCrab app itself needs FDA (you can drag an app icon from Finder
  into the FDA list). When only the Endpoint Security Extension entry is still
  pending, the button is hidden (the extension is already in the list and
  revealing the `.app` bundle in Finder doesn't help).

- **DMG opens with the standard drag-to-Applications install window.**
  The release build script was missing an `/Applications` symlink in the DMG
  staging directory. Finder therefore opened the DMG as a plain folder,
  requiring users to manually drag `MacCrab.app` to their Applications folder.
  The symlink is now included so Finder shows the familiar side-by-side layout.
