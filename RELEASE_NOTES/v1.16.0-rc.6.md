# v1.16.0-rc.6 — KnowledgeC + Quarantine collectors

Plan §13.6 backlog — two of three. FSEvents replay deferred (`/.fseventsd/*.gz` is a specialized binary gzip-compressed log format requiring its own parser).

## What landed

**`com.maccrab.forensics.knowledgec`** — reads `~/Library/Application Support/Knowledge/knowledgeC.db`. Emits `knowledgec.event` per ZOBJECT row: stream name (e.g. `/app/usage` / `/app/inFocus` / `/visit/web`), value string (typically a bundle id or URL), start timestamp (decoded from Mac absolute time), duration. Cap 20,000 events per run. Privacy class metadata.

**`com.maccrab.forensics.quarantine`** — reads `~/Library/Preferences/com.apple.LaunchServices.QuarantineEventsV2`. Emits `quarantine.event` per LSQuarantineEvent row: UUID (cross-referenceable against any file's `com.apple.quarantine` xattr) + originating URL + downloading agent + bundle identifier + timestamp + domain. Cap 5,000 events per run. Privacy class metadata.

Both snapshot-before-parse via `LiveDBSnapshot` since CoreDuet / LaunchServices may have either file open.
