# MacCrab v1.7.8 — Dashboard UX hot-fix

Two reported visual issues addressed.

## Fixed — zombie-sysext banner

The reboot-recommendation banner (orange, shown when 3+ prior MacCrab
sysexts are queued for uninstall on reboot) had three issues:

1. **Translucent** — 12% orange tint over content meant the underlying
   panel bled through; on tall windows it appeared to visually merge
   with the daemon-offline banner stacked above.
2. **No dismiss button** — once shown, only a daemon-driven count
   change could remove it.
3. **Visual overlap** with the top window banner.

v1.7.8:
- Opaque `.regularMaterial` background so nothing bleeds through
- Orange visual cue moved to a 3-pt leading edge stripe
- `×` dismiss button on the right, persisted via `@AppStorage`
- Banner re-appears automatically if a future upgrade adds MORE
  zombies than the count you previously dismissed

## Fixed — sidebar overlays content on narrow windows

`NavigationSplitView`'s default style would collapse the sidebar into
an overlay whenever the window was resized narrow, sliding over the
detail content rather than letting the detail area scroll/wrap.

v1.7.8:
- `.navigationSplitViewColumnWidth(min: 200, ideal: 220, max: 280)`
  constrains the sidebar
- `.navigationSplitViewStyle(.balanced)` keeps both columns visible
  side-by-side; the sidebar now only collapses when explicitly toggled
  via the toolbar, never as a side-effect of window resize

## How to install

Sparkle auto-update / `brew upgrade --cask maccrab` / fresh DMG from
[github.com/peterhanily/maccrab/releases/v1.7.8](https://github.com/peterhanily/maccrab/releases/tag/v1.7.8).

## What you don't need to do

UX-only patches in `MacCrabApp` target. Daemon is unchanged from
v1.7.7. No reboot, no extension re-approval, no data migration.
