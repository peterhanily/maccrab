## MacCrab v1.5.3 — Sysext FDA detection fix

Hotfix for the Full Disk Access banner not clearing after granting FDA to
"MacCrab Endpoint Security Extension." No other changes.

### Fixed

- **FDA banner now correctly clears after granting access to the Endpoint
  Security Extension.**

  Root cause: macOS stores the Endpoint Security sysext's FDA grant in
  `/Library/Application Support/com.apple.TCC/TCC.db` (the system-level
  TCC database, owned by root) rather than the user-level TCC database at
  `~/Library/Application Support/com.apple.TCC/TCC.db`. The v1.5.2 query
  only checked the user-level database and never found the entry.

  Fix: the probe now tries both databases. The system-level database open
  silently fails when the app doesn't have root access — that's expected and
  is handled gracefully. A `LIKE 'com.maccrab.agent%'` clause also catches
  any macOS build where the identifier is stored with a `.systemextension`
  suffix.

- **WAL mtime fallback extended to 30 minutes** (was 5 minutes). When neither
  TCC database is readable — because the app itself hasn't been granted FDA
  yet — the banner fell back to checking whether `events.db-wal` had been
  updated within the last 5 minutes. On quiet systems with no recent events
  this was too short, causing the sysext to incorrectly appear as needing FDA
  even after access was granted. 30 minutes is a more realistic activity window.

628 tests pass. No detection rule changes. No schema changes.
