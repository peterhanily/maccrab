# v1.16.0-rc.16 — FSEvents binary record-stream parser

Closes §13.6 — the FSEvents binary record-stream parsing that v1.16.0-rc.7 deferred is now wired.

## What landed

**`FSEventsRecordParser`** — handles the 1SLD / 2SLD / 3SLD format variants. Decompresses via `gunzip -c` subprocess (Apple Compression framework's raw-deflate handshake is fiddlier than the millisecond shell-out for once-per-log-file work). Walks the record stream extracting null-terminated path + 8-byte event_id + 4-byte flags (+ 8-byte node_id for v2+, + 4-byte reserved for v3).

Decoded flag tokens cover the dominant Apple FSEvents flag set: `Created` / `Removed` / `InodeMetaMod` / `Renamed` / `Modified` / `Exchange` / `FinderInfoMod` / `ChangedOwner` / `XattrMod` / `IsFile` / `IsDir` / `IsSymlink` / `IsHardlink`.

**`FSEventsPlugin`** now emits `fsevents.record` artifacts (capped 1000 per log file) alongside the existing `fsevents.log_file` discovery output. Manifest gains the second output type.
