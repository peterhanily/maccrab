## MacCrab v1.5.5 — Full Disk Access banner: real fix this time

Fourth attempt at fixing the "Full Disk Access is not granted" banner that
wouldn't clear even after granting FDA to both MacCrab and the Endpoint
Security Extension. Apologies for the iteration — the earlier fixes were
all attacking the wrong layer. This one redesigns the detection from first
principles.

### What was wrong

v1.5.2 through v1.5.4 tried to detect the sysext's FDA state by reading
`TCC.db` from the dashboard app's process. That approach cannot work:

- The sysext's FDA grant actually lives in the **system-level TCC database**
  at `/Library/Application Support/com.apple.TCC/TCC.db`, which is mode
  `600 root:wheel`. A non-root process — which the dashboard is — can't
  open it.

- `TCC.db` itself is also TCC-gated under the very Full Disk Access service
  we're trying to test. Granting FDA to the app allows it to read the
  **user's** `TCC.db` (`~/Library/...`), but the sysext doesn't have an
  entry there. So even when both toggles were on in System Settings, the
  app was looking in the wrong database and reporting "sysext needs FDA."

The v1.5.3 change that added the system-level TCC.db path silently failed
on every run (Unix permission denied); v1.5.4's symlink reject and `LIKE`
tightening didn't touch the underlying problem.

### What v1.5.5 actually changes

- **The sysext now probes its own FDA authoritatively.** On every 30-second
  heartbeat, the sysext (running as root, which bypasses neither TCC nor
  `600` Unix perms on its own, but CAN open the system TCC.db because TCC
  allows it once FDA is granted) attempts to `sqlite3_open` and query the
  system `TCC.db`. A successful read means FDA is granted; TCC denying the
  open means it isn't. The result is written into `heartbeat.json` as
  `sysext_has_fda`.

- **The dashboard reads `sysext_has_fda` from the heartbeat as the primary
  signal.** Authoritative, updates within one poll cycle after granting
  FDA. The previous TCC.db probes and WAL mtime heuristic are retained as
  fallbacks for legacy sysexts that haven't written a v2 heartbeat yet.

- **Added a "Dismiss — I've granted access" button on the banner** as an
  escape hatch. Tap it and the banner disappears and stays dismissed across
  launches (persisted to UserDefaults). If our detection later confirms
  both principals have FDA, the dismissal is automatically cleared — so a
  future FDA revocation still re-raises the banner.

### Note on upgrade path

The first heartbeat written by the new sysext takes up to 30 seconds after
the sysext restarts. If Sparkle delivers this update while the sysext is
already running, the dashboard app is on v1.5.5 but the sysext may still
be v1.5.4 for a short period (until `sysextd` swaps in the bundled one).
During that window the app falls back to the legacy TCC.db probe and the
manual "Dismiss" button. Once the sysext is refreshed, the heartbeat path
takes over automatically.
