# v1.13.0-rc.6 — Touch ID DEKVault + forensics MCP meta-tools

**Status: internal release candidate.** Not published to Sparkle appcast or Homebrew cask. Source tag only.

First v1.13b RC. Operator surface lands in pieces; this RC covers the two pieces that don't require new UI: Touch ID convenience on the keychain DEK vault, and the seven MCP meta-tools that expose the substrate to AI agents.

## What landed

**Touch ID DEKVault upgrade.** `KeychainDEKVault` now constructs the keychain access-control with `.userPresence` flags (Touch ID OR device passcode) by default. Hardware that supports Touch ID prompts there; everything else falls back to the login password. The previous device-passcode-only behavior is reachable via `KeychainDEKVault(authPolicy: .devicePasscode)` for cases the operator creates with the `--no-biometric` opt-out (CLI surface for that flag arrives in a follow-up).

**Seven forensics MCP meta-tools** wired into `maccrab-mcp`:

| Tool | Purpose |
|---|---|
| `forensics.list_plugins` | List every registered plugin; optional `category` filter (collector / enricher / fingerprinter / analyzer). |
| `forensics.run_collector` | Invoke a Collector against a case; returns artifacts_committed + status + notes. |
| `forensics.search_artifacts` | Search a case's artifacts; filters by content_type / observed_at window / limit. Privacy gate: results capped at metadata-class unless `case.ai_content_allowed=1`. |
| `forensics.get_artifact` | Fetch a single artifact by id (subject to the same privacy gate). |
| `forensics.timeline` | All artifacts ordered by observed_at; default limit 200. (v1.13b extended.) |
| `forensics.explain_case` | Summary: name + created + encryption + AI grant + scheduled-trusted + artifact totals by content_type. (v1.13b extended.) |
| `forensics.posture_findings` | Return findings emitted by the v1.15 posture Analyzer. v1.13b: empty array with note (surface reserved). |

Each tool routes through `CaseManager` + `PluginRunner` + `ArtifactStore` — the same path the CLI uses. AI agents that previously couldn't see the plugin platform can now drive it.
