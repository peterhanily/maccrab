# MacCrab v1.17.6

A usability and control release.

## New

- **Built-in detections are now visible and tunable.** The built-in `maccrab.*` detections — campaigns, behavioral scoring, sequence and graph rules, correlation, forensic scans, and more — now appear in Detection → Rules. Mute one or override its severity; the detection keeps running even when its alert is muted.
- **Richer alerts.** An alert now leads with what actually happened — the triggering process, the full command, the destination, code-signing and hashes — instead of just the rule's generic description. The Events view links forward to the alerts an event triggered.
- **Agent control (MCP).** A Claude/Codex session can tune detection, author rules, and adjust engine config through the MCP server, under three opt-in tiers that are **all off by default** and enabled only from Settings → Agent Control. Every change is audit-logged and routed through the privileged inbox; response actions never auto-execute. Includes a `maccrab` skill and a `/maccrab-control` command.

## Improved

- **Quieter notifications.** The notification floor now defaults to **High** (with a one-time migration off the noisiest setting and a warning when set low), and several developer-routine AI-tool detections were recalibrated so they're recorded without posting a banner.
- **Seamless rule creation.** The rule wizard now compiles and installs a new rule directly into the engine — it appears in the rules list and fires immediately — instead of exporting a file.

## Fixed

- **Event sources recover after an update.** Sources that stalled at 0 ev/s following an in-place update now recover automatically instead of needing a restart.
- **Built-in-rule settings hardening.** The built-in-rule settings file is now read-only to non-root users — it's written only by the engine through the audited control path, so a local process can't mute a detection by editing it directly.
- **Rule validation + docs.** Rule validation now accepts the standard Sigma / MITRE tag namespaces (including D3FEND), a malformed rule identifier was corrected, and the documented rule/test counts were refreshed.

## Upgrade

- **Sparkle:** accept the in-app update prompt and relaunch.
- **Homebrew:** `brew upgrade --cask maccrab`
- **Manual:** download the DMG from the release page and replace MacCrab.app.
