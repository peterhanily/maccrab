# MacCrab v1.21.3

Store-plugin capability plus Forensics and catalog polish.

## Store / community plugins

- **A sandboxed community plugin can now read the files it declares.** It still
  runs fully contained — deny-default sandbox, no full-disk access — but its
  declared, consented read paths are served through a mediated file broker, so a
  community forensic collector actually works on the store lane. Plugin authors
  read declared files through the new `MacCrabPluginKit` API (one call works on
  both the sandboxed and first-party lanes); see the Tier-B README.

## Catalog

- **Installed plugins are recognised.** An already-installed plugin shows
  "Installed · up to date" (or "· update available" with an Update button)
  instead of a fresh Install button.
- **Run right after installing.** The installed-plugin actions include a Run
  button that runs the plugin on this Mac immediately.

## Forensics view

- **Built-in scanners appear immediately** — previously they only showed up
  after you ran another scanner.
- **"Run on this Mac" no longer jars the window** — the plugin inspector
  dismisses cleanly before the scan starts.
- Plainer wording: a refused or errored plugin names the real reason instead of
  internal jargon.

## Notes

No detection-engine rule changes (436 single + 41 sequence + 6 graph). Signed,
notarized, and delivered via Sparkle auto-update.
