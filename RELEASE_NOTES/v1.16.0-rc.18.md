# v1.16.0-rc.18 — CSQLCipherInitGate + LiveDBSnapshot tests re-enabled

Hardening — closes the parallel-init race that v1.13.0-rc.5 / rc.6 / rc.7 / rc.9 each had to paper over with `@Test(.disabled)`.

## What landed

**`CSQLCipherInitGate`** — process-wide lock at `Sources/MacCrabForensics/Storage/CSQLCipherInitGate.swift`. Serializes the `sqlite3_open` + (where applicable) `PRAGMA key` + initial migration window across every SQLCipher consumer. Per-connection thread safety is still `SQLITE_OPEN_FULLMUTEX`'s job.

**`LiveDBSnapshot.snapshot` now wraps its `sqlite3_open` calls in the gate.** Was previously racing with `ArtifactStore.init` and other parallel tests, producing sporadic `SQLITE_MISUSE (21)` or `sqlite3_backup_init failed: unknown database main`.

**Four previously-disabled tests re-enabled:**

| Test | Was disabled in |
|---|---|
| Snapshot produces a sha256-named file under snapshots/ | v1.13.0-rc.7 |
| Snapshot is parsable read-only as a normal SQLite DB | v1.13.0-rc.7 |
| Snapshotting unchanged source twice dedupes via sha256 filename | v1.13.0-rc.7 |
| Snapshot reflects pre-write state when source is modified after snapshot | v1.13.0-rc.5 |

3 of 3 consecutive full-suite runs green (1812 tests / 337 suites). Verified the gate holds under parallel test execution.
