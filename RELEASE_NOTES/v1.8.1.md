# MacCrab v1.8.1 — Trust hardening + AES-GCM + docs

External-review-driven security and credibility hardening. The v1.8.0
external review (May 4) flagged secrets-store gaps, AES-CBC-not-GCM
encryption, response-action documentation, and the website-vs-repo
version drift as the things that most weakened MacCrab's trust surface.
v1.8.1 closes all four.

No daemon behavior changes for normal operation; this release is
defense-in-depth + documentation, not new detection.

## Security hardening

- **AES-256-GCM database encryption.** Sensitive columns now use
  authenticated encryption via CryptoKit. Tampered ciphertexts produce
  a logged warning instead of decrypting to garbage. Pre-1.8.1 AES-CBC
  ciphertexts (`ENC:` prefix) still decrypt cleanly via the legacy path
  — natural migration as rows get rewritten.
- **Shared keychain access group between `.app` and System Extension.**
  Both bundles now declare `79S425CW99.com.maccrab.shared`. Pre-fix the
  sysext couldn't read API keys the dashboard wrote and fell back to
  env-var / plaintext `llm_config.json`. Self-healing migration: the
  first read after upgrade rewrites legacy keychain items into the
  shared group.
- **Database encryption key** now uses the same shared access group, so
  either bundle can decrypt the events.db.

## Trust documentation

Four new docs make MacCrab's trust surface explicit and reviewable:

- **`docs/TRUST.md`** — release verification: signing identity,
  `codesign` / `spctl` commands, Sparkle EdDSA key, ES entitlement
  check.
- **`docs/THREAT_MODEL.md`** — in-scope and out-of-scope attackers
  with defenses + residual risk for each. Acknowledges remaining gaps
  explicitly.
- **`docs/RESPONSE_SAFETY.md`** — defense layers + per-action
  validators + abuse-case test mapping for every response path
  (kill / quarantine / network-block / runScript / TCC / DNS sinkhole
  / persistence guard / supply-chain gate / panic button).
- **`docs/MODULES.md`** — stable / experimental / opt-in classification
  per subsystem. Stable: rule engine, ES collector, alert/campaign
  storage, most outputs, baseline collectors. Experimental: sequence
  engine, campaign detector, AI Guard cluster, response actions, OTLP.
  Opt-in: ultrasonic, TEMPEST, fleet, deception.
- **`docs/COVERAGE.md`** — generated rule-to-MITRE-ATT&CK matrix.
  424 rules across 18 tactic directories, 154 distinct technique IDs.
  Re-emit via `make coverage-doc`.

## Marketing copy

- **README + maccrab.com lede** repositioned: *"open, local-first
  detection and investigation for developers, researchers, and Mac
  security practitioners."* Avoids any framing that implies drop-in
  EDR replacement.
- README adds an explicit positioning paragraph clarifying scope: no
  fleet management, no SOC, single-host.
- Site meta keywords drop "macOS EDR" / "open source EDR".

## Website auto-fetch

- **`scripts/build-release.sh`** writes `release.json` per release with
  version, rule count, test count, DMG sha256, release date.
- **`scripts/publish-release-json.sh`** ships it to the
  `peterhanily/maccrab-site` repo.
- **Site `index.html`** fetches `/release.json` on load and updates any
  element with `data-release="version"` / `"rules"` / `"tests"` /
  `"dmg-url"`. Eliminates the version-drift class of bug — external
  reviewer caught us showing 1.7.12/929 tests post-1.8.0.

## Module status surface

- **`Sources/MacCrabCore/ModuleStatus.swift`** — canonical 41-entry
  catalog with maturity classification per subsystem. Surfaceable in
  the dashboard's About panel and exported to `docs/MODULES.md`.

## UX

- **Security grade hover popover bug** — popover was dismissed the
  instant the cursor left the StatCard, making it unreadable beyond
  the first row. Dual-hover tracking now: trigger OR popover content
  keeps it open, with a 250ms grace period for cursor transit.

## Compatibility

- **No reboot or extension re-approval** required. Same bundle ID,
  same sysext approval.
- **No data loss.** AES-GCM migration is read-forward (old ciphertexts
  still decrypt); keychain access group migration is self-healing.
- **macOS 13 minimum** (unchanged).
- **`maccrabctl` upgrade note:** if the binary in
  `/opt/homebrew/Caskroom/maccrab/<old>/bin/maccrabctl` is stale (cask
  symlink not refreshed), `brew reinstall --cask maccrab` replaces it
  with the v1.8.1 build that knows about the alerts.db split.

## Tests

**983 tests in 197 suites** (was 979). 4 new in `DatabaseEncryptionTests`
covering AES-GCM round-trip, prefix invariant, tamper detection, nonce
randomness, unicode round-trip.
