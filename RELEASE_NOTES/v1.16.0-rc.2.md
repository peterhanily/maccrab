# v1.16.0-rc.2 — BAM parser completes launchd-lite

**Status: internal release candidate.**

Closes the v1.13a-4 deferred sub-slice. `launchd-lite` now inventories `BackgroundItems-v9.btm` (BAM) login items + launch entries alongside its existing plist walk.

## What landed

**`BAMParser`** — pure-function parser for the macOS Background Task Management state file. The format is undocumented (Apple-internal); the parser walks the `NSKeyedArchiver`-encoded plist graph by key-pattern matching, tolerating shifts across macOS 13/14/15 BAM revisions.

Per-record fields:
- `uuid`, `display_name`, `identifier` (bundle id or path).
- `type_raw` (BAM's numeric type) + `type_token` (operator-facing: `launch_agent` / `launch_daemon` / `login_item` / `login_item_privileged` / `application` / `other_<n>`).
- `parent_bundle_id` (for sub-items).
- `url`, `generation`.
- `is_bundle_id` (heuristic: `com.foo.bar` style vs `/path/to/binary`).

**`LaunchdLitePlugin`** wires BAM parsing into its existing collect path. Each BAM entry emits a `launchd.bam_entry` artifact (privacy class metadata). Manifest gains the second output type.
