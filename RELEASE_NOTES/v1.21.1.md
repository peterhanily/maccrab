# MacCrab v1.21.1

A small fixes release: menu bar polish plus security and correctness hardening.

## Fixes

- **Menu bar icon.** The menu bar always shows a single, normally-spaced crab;
  alert and degraded-protection states no longer render a wider, double-width
  icon with a gap.
- **Response-action configuration is admin-gated.** Automatic response actions
  (kill, quarantine, block network) are now only honored from a configuration
  file owned by an administrator account — matching the privilege bar already
  applied to the engine's control channel. A standard, non-administrator local
  account can no longer influence them.
- **DNS collector.** Fixed a file-descriptor double-close on a network-interface
  attach failure.
- **Plugin revocations.** The plugin catalog and revocation list are now always
  fetched fresh, so a newly revoked plugin can never appear trusted from a stale
  cache.

## Notes

No detection-engine rule changes from v1.21.0 (436 single-event + 41 sequence +
6 graph rules). Signed, notarized, and delivered via Sparkle auto-update.
