# v1.13.0-rc.2 — codesign-resolve enricher + Pass 2026-C

**Status: internal release candidate.** Not published to Sparkle appcast or Homebrew cask. Source tag only.

Closes v1.13a-2 of the Mac Context Plugin Platform arc. The platform now ships its first real plugin — a reference Enricher that resolves codesign posture for any binary path. Same SecStaticCode logic that MacCrabCore already uses for events, now reachable as a plugin so dashboard / MCP tools / future Analyzers can consult it on demand.

## What landed

**`com.maccrab.enricher.codesign-resolve`** — the reference Enricher plugin. Takes an `EnrichmentSubject` (event payload, alert payload, or path-on-disk), runs it through `MacCrabCore.CodeSigningCache`, and emits a flat field bag:

- `codesign.signing_status` — `unsigned` / `adhoc` / `developer_id` / `apple` / `invalid` / `unknown`
- `codesign.team_id` — Apple Developer Team Identifier
- `codesign.bundle_id` — declared bundle identifier
- `codesign.notarized` — `true` if stapled ticket present
- `codesign.hardened_runtime` — `true` if `CS_RUNTIME` flag set in the codesign flags

Cached via `CodeSigningCache` (existing actor in MacCrabCore) so re-querying the same path is free after the first lookup. Privacy class `metadata` — exposable to MCP without operator opt-in.

**PluginRunner.runEnricher.** PluginRunner gains a second invocation path. Same lifecycle as `runCollector` (open plugin_invocations row → call enrich → close row), with the kind-check now distinguishing Enricher from Collector. Three new tests cover unknown id / kind mismatch / happy path.

**Pass 2026-C in pre-release-audit.sh.** Every Enricher type under `Sources/MacCrabForensics/Plugins/Enrichers/` must have a paired test file with an idempotency assertion (`fields == fields` across re-runs, or text containing "idempotency" / "byte-identical"). The pass enforces that future enricher authors don't ship a non-pure `enrich(subject, stage)` without documenting the byte-identical contract.

## Tests

1708 total across 318 suites passing. Plugin-platform-specific additions:
- `CodesignResolveEnricherTests` — 4 cases (Apple-signed system binary, unknown path, nil exec path, event subject path resolution).
- `CodesignResolveEnricherIdempotencyTests` — 3 cases proving byte-identical fields across re-runs (the Pass 2026-C invariant).
- `PluginRunnerEnricherTests` — 3 cases (runner happy path, unknown id, kind mismatch).

## What's intentionally NOT in this RC

- Track 1 event-pipeline integration — Sigma rules can call the enricher via the runner / MCP, but the existing exec event flow doesn't yet auto-decorate every event with `codesign.*` fields. That wiring lands with v1.13a-3 when the TCC-lite collector also needs the enrichment hook.
- Other enrichers (threat-intel, geoip, dns-passive-reputation) — backlog per plan §13.7.
