# MacCrab v1.8.0 — Storage discipline, OTLP, dashboard polish

A focused release built around a per-tier storage redesign so heavy event
volume can no longer evict alert and campaign history. Adds OpenTelemetry
output, three new AI-safety rules, dashboard pagination + activity charts,
and a clutch of security and crash fixes.

## Storage redesign

- **Alerts now live in `alerts.db`** (split from `events.db`). One-shot
  startup migration relocates existing alerts; idempotent on re-run.
- **Campaigns continue in `campaigns.db`** (since v1.6.22). v1.8 adds
  retention + size-cap timers for it.
- **Three independent retention budgets** replace the single
  `retentionDays` + `maxDatabaseSizeMB` knob. Defaults:
  - Event firehose: 30 min hot tier, ≤ 200 MB (slider 15 min – 24 h)
  - Alert history: 365 days, ≤ 100 MB
  - Campaign history: 365 days, ≤ 50 MB
- **Adaptive rollup** progressively tightens the events cutoff if the DB
  is over target, with a 15-minute floor (the longest sequence rule needs
  a 10-minute window).
- **Alert-evidence capture is bounded**: ±30s window, top 50 rows per
  alert by severity then proximity. Existing oversize captures are
  trimmed by the rollup sweep.

Legacy `retentionDays` / `maxDatabaseSizeMB` config keys still work —
they fold onto the new shape automatically.

## Detection

- **3 new AI-safety rules**: SKILL.md poisoning install, Claude Code
  project-config RCE (CVE-2025-59536), agent dotfile persistence.
- Severity demotion on three high-FP rules (remote thread injection,
  Python -c suspicious, shell-spawned-by-browser).
- Two long-deprecated rules removed (invisible Unicode in source,
  Trojan source bidi).
- Rule corpus: **425 rules** (382 single-event + 38 sequence +
  internal classifications).

## Output

- **OpenTelemetry Protocol (OTLP) HTTP/JSON** sink. Configurable
  endpoint URL, batch size, flush interval. Joins the existing Splunk
  HEC, Elastic Bulk, Datadog Logs, Wazuh API, S3, and SFTP outputs.

## Dashboard

- **Activity timeline on Overview** — severity-stacked alert bars with
  campaign-fire markers overlaid as flag glyphs above the hour they
  fired.
- **SQL-side event histogram** on the Events tab. Pre-fix the chart was
  built from the 500-row in-memory cache and collapsed to a single bar
  on high-volume hosts. Variable granularity by window (1 min / 30 min /
  6 h / day) so the spread is always visible.
- **Hover tooltips on all histograms** — bar darkens, labelled overlay
  shows time bucket + count (alerts case adds severity breakdown).
- **Security grade hover popover** on Overview. Hovering the Security
  stat card shows points-earned-of-max for every factor, grouped by
  category (System / Runtime / Hygiene), with pass/warn/fail status.
- **AI Guard timeline** — search box, CSV/JSON export, scrollable, and
  spawn-cluster rollup that collapses repetitive same-basename spawns
  within 5 s.
- **Settings → Storage** — three-row panel exposing all per-tier
  retention + cap budgets, with current size readout per tier.

## Pagination + DB-side search

- **Keyset cursor pagination** for Alerts and Events. Load-older button
  walks back through history without re-fetching the recent rows.
- **Database-side search** — Alerts via LIKE on indexed columns, Events
  via FTS5. Replaces the prior in-memory pass that only saw the top-N
  cached rows.
- **MCP tool limits aligned** with declared schema (max 100, default
  20–50). Prior versions silently truncated to 20 rows.

## Security

- **TLS floor + scheme validation** on `StreamOutput` and `S3Output` —
  closes a cleartext-token-leak path when an operator pointed an output
  at `http://` or a custom S3-compatible endpoint without HTTPS.
- **Slack / Teams / Discord / PagerDuty webhooks** now go through the
  same SSRF policy as the generic webhook output.
- **Database encryption key** in Keychain pinned to
  `kSecAttrAccessibleAfterFirstUnlockThisDeviceOnly` so it never roams
  via iCloud Keychain. Same fix applied to `SecretsStore.SecItemUpdate`.
- **`runScript` response action** rejects scripts not owned by `root`
  in an allowlisted directory — closes a local privilege escalation
  where a non-root user could redirect the sysext to run arbitrary
  binaries.
- **Bulk-suppress crash fix** — `String.LocalizationValue` interpolation
  with `Int` arguments now matches the `.strings` `%lld` declarations.

## Defaults

- **Notifications default to critical only**. New installs receive OS
  notifications for critical alerts only; everything else is visible
  in-app. Existing UserDefaults preserved.
- **Event-insert filter** expanded with field-measured noise patterns.
  ~17–25 % volume reduction on dev workloads.
- **Cross-process correlator** ignores SwiftPM `.build/` paths so
  toolchain rebuilds don't generate noisy correlation alerts.

## CLI

- **`maccrabctl rollup`** — manual rollup-sweep trigger. Useful for
  reproducing storage migrations on a captured DB.
- **`make breakdown`** — diagnostic dumping insert-rate, top noisy
  process paths, and table sizes for an `events.db` snapshot.

## Tests

- **979 tests in 197 suites** (was 953). New coverage for the storage
  split (relocator + per-store size-caps), config migration paths,
  and SQL-side pagination cursors.

## Compatibility

- **No reboot or extension re-approval** required. The system extension
  bundle ID is unchanged; sysext activation re-uses the existing approval.
- **No data loss**. The alerts-table relocation copies before dropping
  and is idempotent on partial-failure restart.
- **macOS 13 minimum** (unchanged).
