## MacCrab v1.7.1 — Panel-richness audit (Track 2)

The carry-over Track 2 from v1.6.19 → v1.6.20 → v1.6.21 → v1.6.22 →
v1.7.0. Four primary panels gain the v1.6.17 Threat Intel rebuild
template: search + per-row metadata expansion + per-source health +
multi-view modes for the panels where one list view didn't tell the
whole story.

### Rules panel — RuleBrowser

- Per-rule **fire count + last-fired timestamp + mean exec ms** in
  every row, fed by a new `<supportDir>/rule_telemetry.json` snapshot
  the daemon writes on the heartbeat tick.
- **"Slow only" filter** — show just the rules whose mean execution
  exceeds the 50 ms slow-rule threshold the daemon already logs.
- **"Most fires" sort mode** alongside the existing alphabetical sort.
- **`SLOW` row badge** on rules currently above the threshold.

### Browser Extensions panel

- **Cross-browser search** by name, extension ID, or permission token.
- **Collapsible per-browser sections** with a per-browser
  "N flagged" badge so you can fold a browser's rows out of the way
  while inspecting another.

### Permissions panel — TCCTimeline

- Three view modes via segmented picker:
  - **Timeline** (existing): chronological grants / revocations
  - **Services** (new): per-service group → list of apps with status
  - **Apps** (new): per-app group → list of services with status
- Services and Apps modes pull from a new
  `<supportDir>/tcc_snapshot.json` the daemon writes on the heartbeat
  tick (current state matrix, 0o644 so the dashboard can read it
  cross-process).

### ES Health panel

- **Live event-rate sparkline** — rolling 10-minute window (60 points
  × 10 s refresh) using `Charts.LineMark` + `AreaMark`.
- **Per-event-category breakdown over the last hour** — horizontal
  bar chart of process / file / network / TCC / etc. event counts,
  fed by a new `event_type_counts_1h` field added to the heartbeat
  schema (now v3).
- "Refresh" button stays — the sparkline updates on each press as well
  as on the regular dashboard refresh tick.

### Daemon-side additions

- `RuleEngine.RuleStats` (Codable) — per-rule fire count, total exec
  ns, last-fired Date. Updated on every `evaluate(_:)` call (fired or
  not). `writeTelemetrySnapshot(to:)` + nonisolated
  `readTelemetrySnapshot(at:)` — same temp+rename atomic pattern as
  every other v1.7.x snapshot.
- `TCCMonitor.PublicEntry` + `PermissionSnapshot` (Codable) +
  `writeSnapshot(to:)` + `readSnapshot(at:)`. Public copy-by-value of
  the previously-private `TCCEntry` so the dashboard can decode it.
- `EventStore.eventCountsByCategory(since:)` — indexed query over
  `idx_events_ts_category` returning `[String: Int]`. Used by the
  heartbeat writer to compute per-category counts over the last hour.
- Heartbeat schema → v3. New `event_type_counts_1h` field.
- Three new daemon writes wired into the existing 30 s heartbeat
  cadence in `DaemonTimers`. The body of `heartbeatTimer.setEventHandler`
  is now wrapped in a `Task` so the EventStore query for per-category
  counts can `await` across actor isolation.

### Audit codification

- `pre-release-audit.sh` Pass 7 — primary panel views must declare a
  search state (`@State searchText`/`query`/`filterText`/`searchQuery`)
  AND a drill-down hook (`.sheet`/`.popover`/`NavigationLink`/
  `HSplitView`/multi-view picker). ES Health is permitted to skip both
  on the strength of its sparkline + breakdown + collector list.
  Codifies the v1.6.17 Threat Intel rebuild template as an
  architectural invariant.

### Deferred to v1.7.2

- Drop-count counter on the heartbeat (would require new daemon
  instrumentation outside the EventStore query).
- Collector health registry (replaces the hardcoded list in ES Health
  with daemon-side last-tick / error reporting).
- Rule-engine execution percentiles (P50/P95/P99) — current `meanExecNs`
  is enough for the slow-rule filter; percentiles are nice-to-have.
- Aider / Codex MCP spawn-shape matchers (carry from v1.7.0).
- AI Analysis / Prevention / Package Freshness / Integrations panel
  rebuilds (P1/P2 panels).

### Notes for operators

- The new snapshot files (`rule_telemetry.json`, `tcc_snapshot.json`)
  appear under `/Library/Application Support/MacCrab/` (system) or
  `~/Library/Application Support/MacCrab/` (user). World-readable
  0o644 — same as the existing `agent_lineage.json` and
  `mcp_baselines.json`.
- The heartbeat schema bump from v2 → v3 is backward-compatible: a
  v1.7.0 dashboard reading a v3 heartbeat sees the legacy fields
  unchanged and silently ignores `event_type_counts_1h`.
- Dashboard refresh cadence is unchanged (10 s). Sparkline takes
  ~10 minutes to fill its 60-point window after a fresh start.
