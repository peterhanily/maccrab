## MacCrab v1.4.0 — Quieter. Safer. Friendlier.

v1.4.0 is a broad quality-of-life release. Three months of field data
drove a focused pass across detection noise, stability, and everyday UX.

---

### Much less noise

- **Interactive admin gate.** `ps`, `lsof`, `defaults`, `dscl`, `csrutil`,
  `system_profiler`, `spctl`, and other admin CLI tools no longer fire
  recon alerts when you run them from a terminal. MacCrab walks the
  process ancestry, recognizes Terminal, iTerm, Warp, Alacritty, tmux,
  screen, and other interactive sessions, and drops non-critical
  matches when your shell is the parent.
- **Vendor-worker chains suppressed.** The cross-process correlator now
  ignores chains whose events all sit inside a single vendor's own
  state directory or come from the same process identity. Added
  Adobe, JetBrains, Zoom, 1Password, Firefox, Notion, Obsidian, iCloud
  Drive, Time Machine, Homebrew temp, and ~15 other common noise
  sources.
- **Discovery rules recalibrated.** `lsof_network_enum`,
  `dscl_user_enumeration`, `ioreg_hardware_enum`, `process_listing_by_unsigned`,
  `debugger_evasion_check`, and `bluetooth_scanning_tool` dropped to Low
  severity. These are reconnaissance signals, not attack evidence —
  campaign detection now escalates them only when corroborated by
  follow-on activity.
- **Kill-chain detection filters out Low-severity noise.** Campaign
  detector now counts only medium-and-above tactics when deciding
  whether multiple stages of an attack are present. No more "Multi-Stage
  Attack" alerts from someone running three admin commands in a row.
- **Broken unicode rule retired.** The v1.3.11-disabled
  `invisible_unicode_in_source` rule stays off; the compiler now
  respects `status: deprecated` so retired rules don't ship enabled.

### Stability

- **Three silent data hazards closed.** CampaignStore now rejects
  symlinks on its DB path (matching EventStore and AlertStore). Sysext
  crash-write failures are logged instead of swallowed across 14 call
  sites. All three stores log PRAGMA failures so a failed WAL-mode
  switch produces a diagnostic instead of silent fallback.
- **Data directory resolution no longer silently picks unreadable
  paths.** If both the user and system DBs are unreadable, MacCrab
  surfaces a warning in Unified Log instead of fighting a permissions
  problem in silence.
- **Flaky collector tests tightened.** 10 lifecycle tests now route
  through a `withStartedMonitor` helper that guarantees `stop()` runs
  even if the body throws, removing a class of residual-state flake on
  slow CI runners.

### Dashboard

- **Alert retention control in Settings.** Clear alerts older than 7,
  30, 90, or 365 days with one click. Event records are untouched.
- **Copy as Markdown.** One-click copy on any alert as markdown —
  properly formatted for pasting into tickets, Slack, or incident
  reports. Plain-text Copy Details is still there.
- **Keyboard shortcuts reference.** New "Keyboard Shortcuts" page in
  the Docs tab documents every in-app shortcut in one place.
- **One-click Enable Protection on first launch.** The Welcome wizard's
  final step now activates the system extension directly, collapsing
  the old two-step flow (dismiss wizard → find button → click) into a
  single "Enable Protection" button.

### CLI

- **`maccrabctl rule enable <id>` / `maccrabctl rule disable <id>`.**
  Park a noisy rule on-disk without deleting the YAML or recompiling.
  Send SIGHUP to the detection engine (or relaunch MacCrab.app) to
  pick up the change.

### Enterprise

- **MacCrab.mobileconfig template for MDM deployment** (`deploy/`).
  Pre-authorizes the system extension, grants Full Disk Access to both
  `com.maccrab.app` and `com.maccrab.agent`, and registers the app as
  a managed login item. Customize, sign with your enterprise profile-
  signing cert, and deploy via Jamf / Kandji / Mosyle / Intune. Fleet
  installs no longer require the System Settings approval prompt.

### Release process

- **Release checklist (`RELEASE_CHECKLIST.md`) + automated preflight
  (`scripts/prerelease-check.sh`).** The release pipeline now refuses
  to sign an out-of-sync version (project.yml vs plists vs README),
  a missing or stub `RELEASE_NOTES/v{VERSION}.md`, a broken rule
  compile, or dropped SPM pin hygiene. Warnings flag stat drift and
  low-coverage localizations.

---

### Upgrading

Backwards-compatible. No schema migration, no re-approval of the
system extension. Existing suppressions and dismissed alerts
persist across the update (the v1.3.12 fix for that landed here).

**Field noise reduction:** if you were seeing GoogleUpdater-driven
cross-process chains, Wi-Fi attack alerts on Chrome Helper, or
"Multi-Stage Attack" campaigns from routine admin commands, v1.4.0
makes all of them stop at the source. Existing noisy alerts in the
database stay until you clear them — use the new retention control
in Settings → Detection Engine → Retention.
