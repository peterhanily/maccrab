# v1.16.0-rc.5 — iMessage-metadata collector

Plan §13.1 iMessage-metadata. The privacy proof point: Messages is the most-sensitive surface operators routinely ask about; the metadata-first model lands here and doesn't ship any message body text out of the parser.

## What landed

**`com.maccrab.forensics.imessage-metadata`** — walks `~/Library/Messages/chat.db` with snapshot-before-parse, emits four artifact types (privacy class `personalComms` throughout):

- **`imessage.handle`** — per-handle record: id (phone / email / Apple ID) + service + country + uncanonicalized form.
- **`imessage.thread`** — per-chat record: chat_identifier + service_name + display_name + group_id + `is_group` flag.
- **`imessage.message_meta`** — per-message record: handle_rowid + date (decoded from Mac absolute time, handles both seconds and nanoseconds variants) + is_from_me + is_read + is_delivered + has_attachments + `text_length` (count only, not the text). **Body text never leaves the parser.**
- **`imessage.url_mention`** — for every distinct URL extracted from a message body via regex: url + domain + handle_rowid + message_rowid + is_from_me. The URL itself is the only fragment of message content that crosses into the artifact store.

Demo positioning: "Show me every URL anyone sent me in the last 7 days — no message bodies."
