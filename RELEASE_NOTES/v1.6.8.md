### 3. UI bug fixes

**Allowlist "Manage" count vs content mismatch.** Previously the
button read `AppState.suppressionPatterns.count` (v1 legacy list)
but opened `SuppressionManagerView` which rendered v2 entries from
`SuppressionManager` on disk. Two disjoint data sources, so the
count and dialog contents never matched. Fixed: `AppState` now
tracks `allowlistEntryCount` from the authoritative on-disk
`SuppressionManager`, refreshed on every dashboard poll and after
the manager popover closes.

**Campaign row tap doesn't expand.** Only the header-HStack button
was tappable; clicking the summary text or tactics pills did
nothing. Fixed: the whole collapsed-region VStack now has
`.contentShape(Rectangle())` + `.onTapGesture { onToggle() }`, so
every pixel of the collapsed card toggles expand. Nested buttons
in the expanded detail (Dismiss / Restore / Copy Details) sit
outside the tap block so they retain their own hit regions.
