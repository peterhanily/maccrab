# v1.16.0-rc.4 — Mail collector

Plan §13.2 Mail forensics, v4-reframed: metadata-first, bodies + DKIM deferred. Privacy class `personalComms` — MCP exposure gated by `case.ai_content_allowed`; plaintext cases reject these artifacts at INSERT (Pass 2026-D).

## What landed

**`com.maccrab.forensics.mail`** — walks `~/Library/Mail/` recursively to find every `Envelope Index` SQLite file (V8 / V9 / V10 layouts). Snapshots each, parses the messages table tolerantly (schema varies between Mail versions), emits `mail.message` artifacts per row:

- `sender_address`, `subject`, `message_id`, `mailbox_rowid`, `conversation_id`.
- `date_sent` → `observed_at` (falls back to `date_received` or capture time).
- Common flag bits decoded: `is_read` / `is_replied` / `is_flagged` / `is_junk`.

Cap 10,000 messages per Envelope Index. Snapshot-before-parse via `LiveDBSnapshot` since Mail.app may hold the file open.

## Deferred per plan §13.2 v4 defaults

- `.emlx` body extraction — `includeFullBodies` default false; content-class artifacts.
- DKIM / SPF / DMARC re-verification — 24h DNS cache; opt-in flag.
- Attachment payload extraction — metadata only.
- Per-mailbox topology aggregation.
