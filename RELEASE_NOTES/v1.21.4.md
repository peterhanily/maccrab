# MacCrab v1.21.4

A performance and hardening release: markedly lighter under load, with stronger sensor integrity and broader detection coverage.

## Faster and lighter

- **Much lower CPU and memory under sustained file activity.** High-volume log writers (the common cause of elevated CPU) are now filtered at the kernel level before they ever reach the detection pipeline, and the per-event pipeline itself is significantly cheaper. On a busy machine the engine settles back to idle far more quickly after a burst.
- **Bounded, self-compacting storage.** The event database holds to its configured size, and search-index maintenance moved off the write path — so ingestion stays fast even as history grows.

## Stronger sensor integrity

- **Honest telemetry when the sensor is under pressure.** Dropped-event accounting is now accurate end-to-end, and the engine raises an advisory signal (with a menu-bar indicator) when sustained load degrades coverage — so a blind spot is visible rather than silent.
- **Tamper-evident event and trace storage** with a verify-on-load continuity check.
- **Stable rule profile by default.** The shipped profile runs the vetted rule set; experimental rules are opt-in via `rule_profile: all`.

## Broader detection

- **Background Task Management (BTM) / login-item coverage** — flags launch items added without a matching on-disk plist and untrusted background-item registrations.
- **AI-agent lethal-trifecta correlation** — a graph rule that correlates private-data access, untrusted input, and external egress within an AI-tool lineage.
- **Download provenance** — high-severity credential/exfiltration alerts now carry the delivering application and origin for downloaded payloads.
- Several archive-based rules now also match macOS's `bsdtar` (macOS's `/usr/bin/tar`), closing a coverage gap.

## Fewer false positives

- Trusted-signer verdicts are cache-stable across app self-updates and moved binaries.
- XProtect-outdated, boot-argument, and rogue-profile checks no longer re-alert on an unchanged persistent condition, and re-alert correctly when a cleared condition returns.
- A browser reading its own profile is no longer flagged as credential access; reading another app's credentials still is.

## Upgrading

MacCrab updates in place via the built-in updater, or `brew upgrade --cask maccrab`. macOS 13+ required.
