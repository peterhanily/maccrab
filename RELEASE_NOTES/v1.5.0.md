## MacCrab v1.5.0 — Detection expansion, deep LLM analysis, new CLI commands

### What's new

**37 new detection rules (417 total, 38 sequences)**

The biggest rule expansion since launch. New coverage for:

- **Exfiltration**: rclone cloud sync, AWS/GCS/Azure CLI uploading from
  sensitive paths, Slack/Discord/Teams file upload API abuse, ICMP
  tunneling tools (ptunnel, icmptunnel), and curl/wget posting to public
  paste services (pastebin, transfer.sh, 0x0.st).
- **Lateral movement**: Bonjour/mDNS host discovery via `dns-sd`
  (stealthy internal network enumeration that avoids port-scan signatures),
  and AirPlay mirroring to non-Apple receivers (screen capture vector in
  shared offices).
- **Sequences**: three new multi-step chains — archive then cloud upload,
  LLM API key harvest then exfiltration (CanisterWorm/NX pattern), and
  SDR device launch then outbound transfer (TEMPEST prep chain).

**Deep campaign analysis with extended thinking**

When MacCrab detects a HIGH or CRITICAL campaign spanning 3+ tactics and
you have a Claude Opus 4 API key configured, the investigation now uses
extended thinking — the model reasons step-by-step before producing its
summary. This produces materially better kill-chain attributions and
defense recommendations for complex attacks. All other backends continue
to work as before.

**Two new `maccrabctl` commands**

- `maccrabctl vulns` — shows CVE scanner findings (CVE ID, severity,
  affected app and path, remediation detail). Supports `--hours` and
  `--severity` filters. Vulnerability findings were previously buried in
  the full alert stream.
- `maccrabctl privacy` — shows egress anomaly findings (Bulk Egress,
  Domain Spike, Tracker Contact) with per-process detail. Supports
  `--hours`.

**UEBA weekday/weekend awareness**

Off-hours login detection now accounts for weekday vs. weekend schedules.
A login at 3 AM on a Saturday is scored differently from 3 AM on a
Tuesday. Existing per-user behavioral profiles upgrade automatically.

**Broader DoH evasion detection**

The DoH detector now recognises 28 resolver IPs (up from 16), adding
Cloudflare for Families, Mullvad, ControlD, DNS.SB, and Comodo Secure
DNS alongside the existing Google, Cloudflare, Quad9, OpenDNS, NextDNS,
and AdGuard coverage.

**More privacy tracking domains**

AppPrivacyAuditor now recognises 70+ tracking and analytics domains (up
from 20), covering Amplitude, Heap, Pendo, PostHog, Braze, Customer.io,
Klaviyo, Datadog, New Relic, Dynatrace, Firebase, FullStory, LogRocket,
MouseFlow, CrazyEgg, LaunchDarkly, Statsig, GrowthBook, and more.

### Fixes

- Three supply chain rules with silently-dropped count aggregation
  expressions now fire on single-access (stronger process filters added
  to reduce false positives).
- Two sequence rules corrected for invalid `|not|startswith` /
  `|not|endswith` Sigma modifiers.

628 tests pass. No schema changes. No breaking API changes.
