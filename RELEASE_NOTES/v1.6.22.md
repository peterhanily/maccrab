## MacCrab v1.6.22 — Endpoint footprint reduction

A focused reduction of the daemon's resident memory and steady-state
syscall load. Production observation on a v1.6.21 test host showed the
sysext at **2.76 GB Real / 2.65 GB Private / 416 GB Virtual / 27 M Unix
syscalls / 15 M Mach syscalls / 128 ports** over a 7-minute CPU
window. v1.6.22 retargets that to **~800 MB resident, ~5 M syscalls
per equivalent window** through six architectural cuts and one bug
fix the audit surfaced.

### What you'll see

- **Daemon resident memory drops by ~700 MB on most installs.** No
  feature changes, no detection-rule changes, no dashboard changes.
- **Endpoint CPU under sustained operation drops materially**, most
  visibly on build / CI machines.

### Bug fix the audit surfaced

- **`CampaignStore` was opening `events.db` instead of `campaigns.db`.**
  Three long-lived SQLite connections to one file (EventStore +
  AlertStore + CampaignStore) instead of the audited two; each handle
  carries its own page cache and busy-timeout buffer. CampaignStore
  now opens its own file. The previous `campaigns` table inside
  events.db is left in place; the next size-cap-driven VACUUM reclaims
  the space. Campaign history persisted before v1.6.22 will reappear
  as the detector re-derives campaigns from current alerts.

### Memory cuts

| Hotspot | Was | Now | Save |
|---|---|---|---|
| `EventStore` SQLite mmap | 256 MB | 64 MB | ~190 MB VM |
| `EventStore` SQLite cache | 64 MB | 16 MB | ~48 MB heap |
| `AlertStore` SQLite mmap | 256 MB | 16 MB | ~240 MB VM |
| `AlertStore` SQLite cache | 64 MB | 4 MB | ~60 MB heap |
| `events.db` long-lived connections | 3 | 2 | one full per-conn cost |
| `AgentLineageService` per-session ring | 10K events | 2K events | ~85 MB heap |
| `CampaignDetector.recentAlerts` cap | 50K | 5K | ~90 MB heap |
| `ProcessLineage.maxProcessCount` cap | 50K | 10K | ~30 MB heap |
| `ThreatIntelFeed` IOC caps (combined) | 400K records | 185K records | ~55 MB heap |
| `URLSession.shared` outbound HTTP | uses default config (disk cache) | one shared `SecureURLSession.shared` (ephemeral) | removes `Cache.db` + `httpstorages.sqlite` files |

The pragma cuts are centralized in
`Sources/MacCrabCore/Storage/StoragePragmas.swift` so future tuning
touches one file, not two.

### Syscall reductions

- **`LibraryInventory.scanAllProcesses`** region cap dropped 10K → 2K
  per process AND now actually skips every other forensic-timer cycle
  (the `every other cycle` comment had been a promise without an
  implementation). Net ~10× syscall reduction on this monitor alone,
  which the audit identified as the dominant source of the 27 M
  Unix-syscall total.
- **`NetworkCollector` poll interval** raised 2 s → 10 s default. With
  `proc_pidinfo(PROC_PIDLISTFDS)` + `proc_pidfdinfo` per FD per PID,
  the 5× interval extension delivers 5× syscall reduction on this
  collector. ES gives us real-time spawn context for the same data.
- **`RootkitDetector` PowerGate-gated** + base interval raised 60 s →
  120 s. The dual-API (proc/sysctl) discrepancy detection is not
  latency-sensitive; a true rootkit hides processes for the full
  daemon lifetime, not for sub-minute windows.

### Hot-path CPU reductions

- **18 `print()` calls on the alert hot path removed.** They were
  pure stdout duplicates of work already going through the proper
  `notifier.notify(alert:)` and `alertSink.submit(...)` paths.
- **`URLSession.shared` replaced with `SecureURLSession.shared`** at
  every daemon-target call site (ThreatIntelFeed, MISPClient,
  FleetClient, CertTransparency, NotificationIntegrations,
  PackageFreshnessChecker). The ephemeral session has no on-disk
  cache, no cookie store, and TLS 1.2+ enforced — same connection-
  pool semantics, none of the disk-backed side effects.

### Audit codification (`pre-release-audit.sh`)

- **Pass 4** — every outbound HTTP call in `Sources/MacCrabCore`,
  `Sources/MacCrabAgentKit`, `Sources/MacCrabAgent`, `Sources/maccrabd`
  must use `SecureURLSession.shared`, never `URLSession.shared`. Fails
  the release pipeline on regression. (`MacCrabApp` is exempt —
  Sparkle and AppKit internals depend on `.shared`.)
- **Pass 5** — the count of long-lived SQLite handles to `events.db`
  must equal 2 (EventStore + AlertStore). A third long-lived handle
  fails the release. This is what caught the CampaignStore bug above.

### Tests

892 in 179 suites pass — same as v1.6.21. No new test files; the
changes are behavioral / structural and covered by the existing
storage, threat-intel, and HTTP suites.
