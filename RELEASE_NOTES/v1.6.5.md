## MacCrab v1.6.5 — Eight more field FPs, fixed at the engine

Continuation of the v1.6.x FP-reduction thread. A test-machine soak
after v1.6.4 surfaced eight distinct noise patterns; four are fixed at
the engine level (new ancestor-walk gate), four at the rule level.

### What was firing

1. **TCC Bypass via Process Injection** (HIGH) on GoogleUpdater and
   its `launcher` helper — Chrome's legitimate updater, but its
   binary lives outside `/Applications/` so the existing
   `filter_developer_signed` didn't catch it.

2. **Credential File Access Followed by Network Upload** (CRITICAL) on
   `/usr/libexec/networkserviceproxy` — Apple's own Network Extension
   control daemon. Reads cookie DBs during Safari sync, opens public-
   internet connections via NSURLSession. `filter_system_path` in the
   YAML exists but sequence-engine per-step filter evaluation can
   race with SignerType enrichment.

3. **Process Listing by Unsigned Process** (LOW) on `/bin/ps` —
   Apple-signed platform binary flagged as "unsigned" because
   `PlatformBinary` enrichment wasn't populated on the event.

4. **Defaults Read on Security-Sensitive Domains** (LOW) on
   `/usr/bin/defaults` — same pattern.

5. **SIP Status Queried via csrutil** (LOW) on `/usr/bin/csrutil` —
   same pattern. v1.6.4 scoped the rule to `status`-like commands, but
   `csrutil status` from an admin prompt still fires because the
   interactive terminal gate requires a shell ancestor which the event
   didn't carry.

6. **Rapid System Information Enumeration** (LOW) on `sw_vers`,
   `system_profiler` — same platform-binary enrichment gap.

7. **MDM Enrollment Status Check via profiles** (MEDIUM) — v1.6.4
   added `ParentImage|contains 'GoogleUpdater'`, but real chains are
   Chrome → GoogleUpdater → launcher → GoogleUpdater → profiles.
   `profiles`'s immediate parent is the *second* GoogleUpdater; Sigma
   filters only see the direct parent.

8. **Hidden File Created in User Directory** (LOW) on Logitech
   Options+ agent — vendor binaries rename across releases so an
   `Image|contains 'logioptionsplus'` filter is brittle.

### Fixes

#### Engine-level

- **New NoiseFilter Gate 6: auto-updater ancestor walk.** Walks the
  full ancestor list, dropping non-critical matches when the subject
  or any ancestor is a known auto-updater. Catches (7) regardless of
  chain depth, and provides a backstop for (1), (2), (8) whenever
  the process tree includes an updater.

- **`isKnownBenignProcess` split into two helpers.** The old single
  helper mixed Apple-system-daemon paths with updater paths; using
  it in ancestor-walk filtering swept Terminal.app in as "benign"
  and would silently kill detection for anything launched from a
  terminal. `isAutoUpdater` is the narrower variant used by Gate 6.
  Campaign detector keeps using the broader `isKnownBenignProcess`
  where "is this an OS component or updater?" is the right question.

#### Rule-level

- **`tcc_bypass_via_injection.yml`:** added `filter_auto_updater_image`
  (Image|contains GoogleUpdater / Sparkle.framework / Keystone /
  Microsoft AutoUpdate) and `filter_auto_updater_endswith` (Image
  endswith /GoogleUpdater, /Autoupdate, /launcher).

- **`credential_theft_exfil.yml` cred_read step:** added
  `filter_apple_daemons` with 16 well-known Apple daemon basenames
  (networkserviceproxy, nsurlsessiond, trustd, apsd, accountsd,
  identityservicesd, cloudd, bird, fileproviderd, CommCenter,
  keychainsharingmessagingd, AMPDeviceDiscoveryAgent,
  SafariBookmarksSyncAgent, rapportd, parsecd, PasswordsKit).

- **Discovery rules:** `process_listing_by_unsigned.yml`,
  `defaults_read_sensitive.yml`, `system_enumeration_burst.yml`,
  `csrutil_status_check.yml` all carry a new `filter_system_path`
  anchoring on `/bin/`, `/sbin/`, `/usr/bin/`, `/usr/sbin/`,
  `/usr/libexec/`, `/System/`. Belt-and-braces with `filter_platform`:
  if PlatformBinary enrichment is missing, the hard path anchor
  still suppresses. Safe because these paths are SIP-protected on a
  healthy system.

- **`hidden_file_created.yml`:** added `filter_hw_vendor_bundle`
  anchoring on `/Library/Application Support/` + vendor subdir
  (Logitech, Razer, Elgato, Corsair, SteelSeries, Blackmagic) plus
  `/Applications/Logi Options+.app/` and `/Applications/Logitech*`.
  More durable than basename matching since vendor binaries rename
  frequently.

### Regression lock

New `AutoUpdaterAncestorGateTests` suite (6 tests):

- GoogleUpdater as subject process suppresses non-critical matches
- `profiles` under 4-deep GoogleUpdater chain suppressed
- Sparkle Autoupdate ancestor suppresses child-process matches
- Critical matches under GoogleUpdater still fire (counter-test)
- Non-updater ancestor chain does NOT trigger gate (counter-test)
- `isAutoUpdaterOrAncestor` public API contract

The counter-tests are the important half — prove that Gate 6 isn't
a universal kill-switch and that a new over-broad ancestor entry
would immediately fail the "unknown tool spawned from Chrome still
alerts" case.
