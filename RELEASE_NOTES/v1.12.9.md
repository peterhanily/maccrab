# MacCrab v1.12.9 — dashboard layout patch

v1.12.9 is a single-purpose patch release on top of v1.12.7.
SwiftUI layout code only — no schema migrations, no new
entitlements, no new external network paths, no rule changes.

If you're on **v1.12.7**, Sparkle in-place upgrade works directly:
open MacCrab.app → **Check for Updates…** → **Install Update** →
relaunch when prompted.

## What was broken

The dashboard window could be dragged narrower than its content's
natural width, which produced two visible symptoms on the menubar
window:

1. **Top bar icons unreachable.** The window's outer minimum
   (950 × 600) was smaller than the dashboard's inner content
   minimum (1280 × 800). When the window was dragged into the
   gap between those two values, the top bar's HStack forced
   itself to 1280 pt regardless of the actual window size,
   pushing the trailing icon cluster (theme toggle / Help /
   Settings) past the window's right edge.

2. **Detail inspector overflow.** Selecting an alert, rule,
   package, or trace opened a 340 pt detail inspector via an
   HStack that pushed the table beside it. At any window width
   where the table's natural minimum plus the inspector plus the
   nav sidebar exceeded the window, the trailing edge clipped —
   either the rightmost table columns or the inspector itself
   were partially hidden.

## What's fixed

### Single source of truth for the window minimum

The outer window minimum and inner content minimum are now a
single value (1180 × 760) declared once on the `WindowGroup`'s
`V2RootView`. The duplicate `.frame(minWidth: 1280, minHeight: 800)`
inside `V2DashboardShell` is removed, so the layout's reported
size always agrees with the window's enforced minimum and the
top bar's icon cluster can never be pushed off-screen.

The icon cluster on the right of the top bar also picks up a
`.layoutPriority(1)` so future layout changes (long
localizations, accessibility text scaling) can't squeeze it
out either.

### Detail inspector now floats instead of pushing

The four workspaces that render a detail inspector — Alerts /
Detection / Intelligence / Investigation — moved from an
`HStack [content | inspector]` push-layout to a
`ZStack(alignment: .topTrailing)` overlay. The inspector floats
over the rightmost ~340 pt of its workspace with a soft left-
edge shadow and a slide-in/out transition driven by the
selection. The table beside it keeps its natural width and the
overall dashboard layout stays stable as the user opens and
closes the inspector.

| Workspace | Inspector |
|---|---|
| Alerts | Alert detail |
| Detection | Rule detail |
| Intelligence | Package detail |
| Investigation | Trace detail (+ slim re-open rail) |

The Investigation workspace's slim "Show details" re-open rail
gets the same overlay treatment so toggling the trace inspector
no longer reflows the graph canvas underneath.

## Compatibility

- macOS 13.0+ (same as v1.12.7).
- Endpoint Security entitlement, code signing identity, and
  notarization profile unchanged from v1.12.7.
- No daemon, agent, or storage changes — the system extension
  binary is rebuilt only to keep the bundle version consistent
  with the app.
- Sparkle EdDSA appcast key unchanged.
