## MacCrab v1.6.6 — AI Suite: six new services, one release

The biggest AI push in the codebase yet. Six independent services
land in this release, every one covered by unit tests, every one
feature-flaggable, every one designed around MacCrab's unique kernel
vantage: **what agents actually do on the machine**, not just what
they say to the model.

### 1. Semantic Alert Clustering

`AlertClusterService` (`Sources/MacCrabCore/Detection/`) groups alerts
by a stable `ruleId::processName` fingerprint. The v1.6.5 noise-soak
run produced 13 raw alerts across 6 hours; those collapse to 10
clusters, 3 of which are 2-alert pairs ready to triage or suppress in
one click.

LLM rationale is an on-demand second pass — idle dashboard polls never
hit the backend. 11 unit tests lock fingerprint stability,
max-severity roll-up, chronological ordering, and the v1.6.5-pattern
baseline.

### 2. Multi-Model Consensus

`LLMConsensusService` (`Sources/MacCrabCore/LLM/`) fans out a
classification prompt to N configured backends in parallel, parses
each response against an allowed-label set, and declares consensus
only when ≥ threshold backends agree on a non-inconclusive label.
Per-backend timeouts prevent the slowest from gating the result. 17
unit tests cover label parsing edge cases (markdown wrappers, colon-
preambles, empty responses), tiebreaks, threshold clamping, and the
directive injection that constrains each backend's output shape.

### 3. Triage Auto-Disposition

`TriageService` (`Sources/MacCrabCore/LLM/`) asks the LLM to propose
one of `suppress / keep / escalate / inconclusive` for a single alert,
with a one-sentence rationale. Cluster size and optional 24-hour
volume are passed as context so recurrence can influence the
disposition. Output is advisory — no auto-action. 12 unit tests cover
the prompt template (all four labels named, MITRE facts carried,
similar-count included), the DISPOSITION/REASON parser (markdown
bullets, off-format fallbacks, rationale truncation), and all four
labels round-tripping through the parser.

### 4. MCP Server Behavioral Baseline

`MCPBaselineService` (`Sources/MacCrabCore/AIGuard/`) fingerprints
each MCP server by the file basenames, DNS domains, and child-process
basenames it touches at runtime. Servers begin in `learning` for the
first N observations AND first M seconds (both gates required), then
promote to `enforcing` — any observation outside the fingerprint
emits a `BaselineDeviation`.

Complements the existing `MCPMonitor`: config drift ("a new server
was added") was already caught; runtime drift ("this server is now
calling evil.ru") was the gap. Prompt-injectable MCP servers are the
emerging attack surface for Claude Desktop, Cursor, Claude Code,
Continue, and VS Code agent hosts. 9 unit tests cover dual-gate
promotion, multi-field deviation on a single observation, domain
normalization (eTLD+1 collapse, case folding, dot stripping), and the
reset semantics.

### 5. Agent Data Lineage

`AgentLineageService` (`Sources/MacCrabCore/AIGuard/`) weaves LLM API
calls together with process spawns, file reads/writes, network
connections, and alerts into a single chronological timeline keyed
by the AI tool's root PID. This is the headline feature —
**MacCrab's unique answer to "what did the agent actually do on my
machine."**

Per-session ring buffer (10,000 events default), LRU session
eviction (32 sessions default) so memory is bounded. 9 unit tests
cover session lifecycle (start/end/duplicate-start), chronological
reassembly regardless of insert order, kind-count roll-up, windowed
query trimming, ring-buffer oldest-drops-first, and LRU eviction
when the session cap is hit.

### 6. Agentic Investigation Loop

`AgenticInvestigator` (`Sources/MacCrabCore/LLM/`) runs a bounded
multi-round loop against a `CampaignDetector.Campaign`. The LLM can
emit tool calls (`describe_rule RULE_ID=…`, `alert_descriptions
RULE_IDS=…,…`, `process_children PROCESS_PATH=…`) that the service
answers from local state; after up to 3 rounds (or when the LLM
emits a `VERDICT:` block), the investigation terminates with a
structured `InvestigationReport` carrying verdict, summary, up to 5
findings, and a recommended analyst action.

Advisory only — the investigator never executes commands, never
makes outbound calls, never modifies state. Tool-call fan-out is
capped at 5 per round. 12 unit tests cover VERDICT/SUMMARY/FINDING/
RECOMMEND parsing (markdown tolerance, missing-section defaults, all
four verdicts round-tripping), the three tool-call dispatchers, the
fan-out cap, and campaign evidence formatting.

### Numbers

- **719 tests pass (up from 649)** — +70 new tests across six suites.
- Six new files in `Sources/MacCrabCore/` (Detection/, LLM/, AIGuard/).
- Six new test files in `Tests/MacCrabCoreTests/`.
- Zero modifications to existing public APIs. Each service wires in as
  an opt-in dependency, feature-flaggable via config.

### What's next (v1.6.7 and beyond)

This release is the foundation. Dashboard UI surfaces for clustering,
agent timelines, and the investigator land in v1.6.7, along with
EventLoop wiring so MCP baselines and lineage events populate
automatically from live ES events. The services themselves ship now,
locked by unit tests, so dogfooding the API shape and catching
anything that's off is the v1.6.6 priority.
