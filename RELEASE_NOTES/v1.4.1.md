## MacCrab v1.4.1 — Readable update notes + suppression diagnostics

Two tight follow-ups to v1.4.0.

### Fixed

- **Update sheets actually render formatted now.** v1.4.0's release
  notes came through Sparkle as a wall of raw Markdown — `**bold**`
  and `### headings` showed up as literal characters. Sparkle's
  `<description>` field is HTML, not Markdown. The release pipeline
  now converts `RELEASE_NOTES/vX.Y.Z.md` to proper HTML before
  publishing the appcast entry, so headings, lists, code spans, and
  links render correctly inside the update prompt. GitHub release
  pages keep consuming the same file as Markdown.

### Added

- **Suppression persistence diagnostics.** Save and load paths for
  dismissed alerts and suppression patterns now emit a diagnostic
  trail to Unified Log so a future regression leaves evidence:

  ```
  sudo log show --subsystem com.maccrab.app \
    --predicate 'category == "ui-state"' --last 1h
  ```

  If suppressions come back unsuppressed after an update, running
  that command shows the file path that was read, how many entries
  were loaded, and whether a write subsequently succeeded.

- **One-shot migration on first launch.** If v1.4.1 reads your
  suppression file from the legacy location (pre-v1.3.12
  `dataDir`), it immediately rewrites it to the stable user-home
  location. Next launch finds it there on the first try — no
  fallback needed.

---

Backwards-compatible with v1.4.0. No schema changes, no re-approval
of the system extension. 597 tests pass.
