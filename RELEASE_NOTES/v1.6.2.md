## MacCrab v1.6.2 — Dashboard looks and feels like a Mac app now

Visual + polish release. The dashboard now tracks maccrab.com's palette
(dark-first with warm orange accent), the status-bar icon is a proper SF
Symbol instead of an emoji, and the About panel links straight to the
website. Plus a handful of smaller correctness and performance fixes
surfaced by three specialist review agents.

### Theme: matches maccrab.com

New `MacCrabTheme` type ports the site's CSS custom properties verbatim —
same hex values for base / elevated / card backgrounds, borders, text
tints, accent orange (with its ghost / hot / dim variants), and severity
palette. Light and Dark Mode pairs are provided so the app respects the
system appearance setting.

`.tint(MacCrabTheme.accent)` cascades the brand orange into every native
control: buttons, toggles, links, progress views, sliders, date pickers.
No per-view plumbing.

### Statusbar icon: proper SF Symbol

Template-rendered `shield.lefthalf.filled` (healthy) or
`shield.lefthalf.filled.trianglebadge.exclamationmark` (degraded). Unlike
the previous emoji 🦀, the icon now adapts to light / dark menu bars
automatically and renders crisply at every scaling factor. Severity flash
during a live alert uses palette-rendered red / orange system shields.

### About + Help

- **About MacCrab** (⌘ in app menu) — custom panel with version, build,
  `© 2026 CaddyLabs`, and a clickable **maccrab.com** link right in the
  credits.
- **Help menu** — new entries: "Visit maccrab.com", "MacCrab Documentation",
  "Report an Issue…". Standard Mac convention; no more hunting through
  sidebar views for a link.

### Info.plist completeness

Added `LSApplicationCategoryType = public.app-category.utilities`,
`NSHumanReadableCopyright`, `NSFullDiskAccessUsageDescription`, and
`NSLocalNetworkUsageDescription`. System Settings → Privacy now shows
meaningful explanatory strings for every permission MacCrab requests.

### Dark Mode + performance fixes

- **Alert popover no longer flashes bright white in Dark Mode.** Was using
  hardcoded `NSColor.white`; now uses `.windowBackgroundColor`.
- **Hardcoded RGB severity colors gone** from Overview; flows through
  `MacCrabTheme.severityCritical` / `.severityHigh` / `.ok`.
- **`eventsPerSecond` no longer re-publishes on every poll when the value
  hasn't changed** — full-app view invalidation on a quiet system drops
  from "every 10 s" to "only when the number actually changes."
- **Alerts search is snappier at scale.** Rewrote the filter pipeline as
  a single-pass lazy filter; eliminated the intermediate `AlertViewModel`
  array that was being reallocated on every keystroke.
- **10-second poll now pauses when the dashboard window is hidden.**
  `.onChange(of: scenePhase)` ties the timer lifecycle to window visibility,
  so a backgrounded MacCrab doesn't keep hitting SQLite.
- **Reduce Motion respected** in CampaignView and AIAnalysisView
  expand/collapse (was previously unconditional).

### Under the hood

- Migrated one `.onAppear { Task { await ... } }` to `.task { await ... }`
  so the async load auto-cancels on view dismount.
- Alert-popover show path converted from `DispatchQueue.main.async` to
  the structured-concurrency equivalent.
- `createStatusBarItem` gained explicit `@MainActor` annotation.

---

638 tests pass. No rule changes, no schema changes. This is the first
release where the dashboard is intentionally visually aligned with
maccrab.com's branding.
