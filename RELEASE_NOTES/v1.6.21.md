## MacCrab v1.6.21 ŌĆö Surface completion + comprehensive audit pass

A small-features-zero release with a big internal upgrade. Three

### What you'll see in the dashboard

- **Pending Actions show up as alerts.** v1.6.20 added
  `requireConfirmation` to gate destructive auto-actions (kill /
  quarantine / blockNetwork) but the gate was invisible ŌĆö operators
  had no way to see what was waiting. Now: when an action is gated,
  MacCrab emits a low-severity informational alert
  (`maccrab.pending-action.kill` etc.) with the rule ID, target
  process, and original alert reference. The alert appears in the
  AlertsView; click into it and use the existing manual-action
  buttons to fire it, or suppress to dismiss. Same surface, finally
  wired.
- **"Test Connection" button in Settings ŌåÆ AI Backend.** Verify your
  Ollama URL or Claude/OpenAI/Mistral/Gemini API key works without
  firing a real alert. Inline status: green check on success, red X
  with a reason on failure ("Missing API key", "Backend unreachable
  ŌĆö check Ollama URL"). Resets on any config edit so a stale OK
  doesn't lead you astray.

### Internal hardening (audit-driven)

A five-axis review of every change shipped in v1.6.0ŌĆōv1.6.20:
performance, security, stability/correctness, functionality
regression, UX/accessibility. Findings were triaged by severity and
fixed pre-push.

**12 fixes you don't see, but they make MacCrab more correct:**

- TOCTOU race fixes in two EventLoop dedup paths that could let
  duplicate alerts slip through under sustained load. The v1.6.19
  AlertDeduplicator atomic-method work was complete; these two call
  sites still used the legacy split pair. Now consistent.
- 23 sites that fired notification banners for alerts that
  AlertSink correctly suppressed as duplicates. The notification
  count looked unchanged while the store was correctly de-duped.
  Notifications now only fire on actually-inserted alerts.
- IPv6 CIDR matching in `SafeBlockableIP` ŌĆö pre-fix you could PF-
  block `2001:4860:4860::8889` (one byte off Google DNS
  `::8888`) and silently break your IPv6 DNS. Now Cloudflare /
  Google / Quad9 / OpenDNS IPv6 prefixes plus loopback / link-local
  / multicast are CIDR-checked, not exact-match.
- MISP feed validator gap ŌĆö `MISPClient.fetchCategorized` extracted
  IOC values without running them through the v1.6.18 validators
  custom imports use. A compromised MISP server could push
  `127.0.0.1` as an IP. Closed.
- `SystemPolicyMonitor` 5-min poll now respects PowerGate (battery
  / thermal throttling). Pre-fix `csrutil status` and `xprotect -v`
  shelled out at full cadence even on low-power mode.
- `AlertDeduplicator.normalizePath` regex pre-compile (~9 % CPU
  saved at sustained load), `SafeQuarantinePathValidator` regex ŌåÆ
  string slicing (~10├Ś faster per call), `SafeBlockableIP` gateway
  resolution cached with 30 s TTL.
- `ResponseEngine.executionLog` capped at 50 K (was unbounded under
  sustained action firing).
- `SettingsView` debounced webhook sync now correctly handles
  `Task.sleep` cancellation (pre-fix `try?` swallowed
  `CancellationError` and partial writes were possible).
- `ResponseActionsView` "Saved. Daemon reloaded." banner now tells
  the truth ŌĆö checks `pkill` exit code and shows "Daemon will
  reload on next start." when the sysext isn't reachable.
