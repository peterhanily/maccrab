## A. Code quality + tests

**Suite status**: 1863 tests in 345 suites, all passing after `make compile-rules` populates `/tmp/maccrab_v3`.

**HIGH severity finding**: `Tests/MacCrabCoreTests/RuleTestHelpers.swift` calls `python3 compile_rules.py` with `standardOutput = .nullDevice` and `standardError = .nullDevice`. When compilation fails or `/tmp/maccrab_v3` is stale, **180+ tests fail with no compiler diagnostic**. Apparent failure mode: "5 test runs in a row failed; 126 issues" — actual cause: rule compilation silently crashed.

Fix in rc.2 polish: capture stderr in a buffer; assert `proc.terminationStatus == 0`; surface the stderr if any rule-dependent test fails. One-paragraph fix; massive debuggability win.

**LOW**: 4 pre-existing Swift 6 strict-concurrency warnings (DaemonSetup / AppleScriptRuntimePlugin / MailBodiesPlugin / SafariLitePlugin / ArtifactStore / maccrab-mcp). Pre-existing, not from rc.1. Track as v1.17.0-rc.6 polish.

**MEDIUM**: rc.1 CLI additions (`ScanCommands`, `EvidenceCommands`, the new `plugin` subcommands) have **zero unit test coverage**. Smoke tests only. Add a `maccrabctlTests` target in rc.6 polish.

## C. Customer first-launch journey

**What exists**: `WelcomeView` — 3-step wizard (Language → "What is MacCrab" features → Setup Checklist). Triggered when `@AppStorage("hasCompletedSetup") = false`. Default landing after dismiss: Overview workspace.

**Gaps**:
1. Welcome wizard mentions detection ("Real-Time Detection", "AI Safety", "Active Prevention", "Privacy First") but **zero mention of forensics / plugins / scans**. Operator finishes onboarding with no awareness those exist.
2. Overview workspace probably doesn't surface a scans tile. Operators may never discover the Forensics workspace organically.
3. Existing operators upgrading from v1.16 will not see WelcomeView (`hasCompletedSetup = true`) — they get rc.2's UI shift with no callout.

**Fix in rc.6 polish**: add a fourth step to the welcome wizard for new operators ("What can I scan? — start your first scan from Forensics"), plus a one-time "What's new in v1.17" callout for upgraders.

## D. rc.2 dashboard rework risk

Three state-persistence vectors to handle:

1. **`v2.dashboard.workspace`** UserDefaults key — holds raw workspace value like `"investigation"`. Rename to `"forensics"` silently fails (falls back to `.overview`).
2. **`v2.dashboard.selectedTabs`** UserDefaults — dict mapping workspace → last-selected-tab. Tabs renamed (`investigationForensicsCases` → `forensicsScans`) become silent no-ops.
3. **`v2.dashboard.recents`** UserDefaults — recent (workspace, tab) pairs. Stale entries.
4. **`maccrab://investigation/forensicsCases`** deep links — break with no redirect.

**rc.2 mitigation**: add legacy-value remap in `V2DashboardState.load()`. Write a migration table:

```
investigation                     → forensics
investigationForensicsCases       → forensicsScans
investigationForensicsPlugins     → forensicsPlugins
investigationForensicsTierB       → forensicsPlugins   (merged)
investigationForensicsArtifacts   → forensicsEvidence
investigationForensicsFindings    → forensicsScans  (findings panel inside scan detail)
```

Plus a deep-link redirect in `V2DeepLink.parse()` that maps old paths to new.

Tests: extend `V2DashboardStateTests` + `V2DeepLinkTests` with the migration table.

## E. Strategic gaps — drive-up-a-notch ideas

Five gaps surveyed. Two go into v1.17, three documented for future.

### Goes into v1.17

1. **Compliance preset scans (rc.3)** — add "Compliance check" as a fifth reason on the "Start a scan" wizard. Maps existing built-in plugins to NIST 800-53 / CIS macOS Benchmark controls. Output: a scan whose summary panel shows control coverage. Unlocks the enterprise / consulting market without significant new collector work.

2. **Plugin author scaffold CLI (rc.4)** — `maccrabctl plugin author init <plugin-name>` scaffolds a new author repo with:
   - `manifest.json` template
   - Sigstore-ready `release.yml` GitHub Actions workflow
   - `Tests/` skeleton
   - `README.md` with the publisher checklist
   - Pre-populated `.github/workflows/` for reproducible builds
   Lives in `maccrabctl plugin author` subgroup. Bootstraps the rave ecosystem.

3. **"What's new in v1.17" first-launch callout (rc.6)** — one-time modal for upgraders explaining the Forensics workspace + Plugins catalog. Triggered by `@AppStorage("seenV17ForensicsCallout") = false`. Operator dismisses; never sees again.

### Standalone v1.18 candidates

4. **Scan diffing / drift detection** — "Compare today's scan to last week's." Forensics-shaped. Shows new persistence, new TCC grants, removed evidence. Needs scan history depth first; standalone v1.18 chapter.

5. **AI investigation assistant in dashboard** — surface the existing LLM backends as a "Ask MacCrab about this finding" UI. Already-wired LLMService; just needs a dashboard widget. Orthogonal to IA work.

### Future / mentioned for record

- Mac fleet / multi-Mac view (fleet collector exists; no dashboard surface yet)
- Public anonymized threat intel feed
- macOS Spotlight integration (CoreSpotlight) for evidence search
- Headless CI scan mode (`maccrabctl scan run --headless --report=junit.xml`)
- OpenTelemetry evidence exporter (extension of existing OTLP events output)
