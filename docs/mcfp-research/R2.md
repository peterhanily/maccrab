# MCFP R2 — Cross-Mac Stability + Imposter Validation

**Status:** prep complete. Execution pending Mac #2 corpus collection.

Plan §6.4 R2. Determines whether MCFP v1's static-only fingerprint family is stable enough to ship the conditional `com.maccrab.fingerprinter.mcfp` plugin + paired enricher (v1.15 deferred).

## Ship criteria

Plan §6.4 R2 + statistical-layer additions:

- **Stability:** at least three of [`arch`, `lc`, `cs`, `ent`] show ≥95% same-binary stability across two Macs **AND** Wilson 95% CI lower-bound on each of those three is also ≥95% (so we don't accept a noisy 19/20 = 95% from too-small a corpus).
- **Separation:** ≥80% of cross-family pairs produce a different `cs` or `ent` component, AND the Wilson 95% CI lower-bound on that fraction is ≥80%.
- **Imposter:** an unsigned binary at a known path produces divergence in `cs` + `ent`.

If R2 passes → ship `com.maccrab.fingerprinter.mcfp` + `com.maccrab.enricher.process-fingerprint`. The posture Analyzer's `posture.fingerprint_drift` finding becomes active.

If R2 fails → public failure write-up + posture Analyzer ships without `posture.fingerprint_drift`. Future MCFP iteration revisits.

## Tooling

Four primitives in `Sources/MacCrabForensics/MCFPResearch/`:

| File | Purpose |
|---|---|
| `CorpusCollector.swift` | Walks a directory, fingerprints every Mach-O, writes JSONL to a corpus file. |
| `CorpusDiff.swift` | Compares two corpus files, reports per-component stability + a R2 verdict. |
| `ImposterHarness.swift` | Copies a target into a sandbox dir, strips its signature, fingerprints both, reports divergence. |
| `MCFPStatistics.swift` | Wilson score 95% CI + different-family separation. The CI lower-bound is the conservative ship gate that guards against noisy small-N results. |

Wired through `maccrabctl mcfp`:

```bash
maccrabctl mcfp corpus collect --target <dir> --output <jsonl>
maccrabctl mcfp corpus diff   <corpus-a.jsonl> <corpus-b.jsonl>
maccrabctl mcfp imposter --target <binary-path>
maccrabctl mcfp separation <corpus.jsonl>
```

`corpus diff` now reports Wilson 95% CI per component alongside the point estimate. `separation` is the new R2 dimension: across all cross-family binary pairs in a corpus, what fraction produce different `cs` or `ent`? Verdict PASS when the Wilson 95% lower-bound on that fraction is ≥ 80%.

## Execution recipe

### 1. Corpus collection on Mac A

```bash
maccrabctl mcfp corpus collect \
    --target /Applications \
    --output ~/mcfp-research/corpus-mac-a-applications.jsonl

maccrabctl mcfp corpus collect \
    --target /usr/bin \
    --output ~/mcfp-research/corpus-mac-a-usrbin.jsonl

maccrabctl mcfp corpus collect \
    --target /System/Applications \
    --output ~/mcfp-research/corpus-mac-a-systemapps.jsonl
```

Expected output: ~500-2000 binaries per `/Applications`, ~600-900 per `/usr/bin`, ~80-200 per `/System/Applications`.

### 2. Corpus collection on Mac B (separate hardware)

**Identical target set.** Same macOS major version recommended for the first measurement; cross-version comparison is a follow-up experiment.

```bash
maccrabctl mcfp corpus collect \
    --target /Applications \
    --output ~/mcfp-research/corpus-mac-b-applications.jsonl
# ... etc, identical to Mac A
```

### 3. Diff

Move both corpus files to one Mac. Run:

```bash
maccrabctl mcfp corpus diff \
    ~/mcfp-research/corpus-mac-a-applications.jsonl \
    ~/mcfp-research/corpus-mac-b-applications.jsonl
```

Output: per-component stability percentages + a structured verdict.

Repeat for `usrbin` + `systemapps`.

### 4. Imposter

```bash
maccrabctl mcfp imposter --target /usr/bin/curl
maccrabctl mcfp imposter --target /Applications/Safari.app/Contents/MacOS/Safari
maccrabctl mcfp imposter --target /System/Applications/Notes.app/Contents/MacOS/Notes
```

Expected: `cs` divergence on every target; `ent` divergence on entitlement-bearing targets (Safari, Notes); `ent` parity on entitlement-less targets (`/usr/bin/curl`).

Run on the developer machine (single Mac, no cross-Mac comparison yet):

```
$ maccrabctl mcfp imposter --target /usr/bin/true
Original:  mcfp1/static/universal/238807fd6f9f/e0c0c66ca8f5/e3b0c44298fc
Imposter:  mcfp1/static/universal/238807fd6f9f/0f8b4319d79f/e3b0c44298fc

Divergence:
  arch —
  lc   —
  cs   ✓     (codesign team_id stripped)
  ent  —     (true has no entitlements either way)

R2 imposter verdict: PARTIAL
```

The PARTIAL verdict on `/usr/bin/true` is the expected shape: entitlement-less binaries don't produce `ent` divergence because the entitlement key set was empty both times. A target like `/usr/bin/curl` or `/Applications/Safari.app` carries entitlements and will produce full PASS verdicts.

Corpus collection at `/usr/bin` (Mac A only):
```
$ maccrabctl mcfp corpus collect --target /usr/bin --output /tmp/corpus.jsonl
Collected: 658 binaries fingerprinted, 263 skipped (non-Mach-O or unreadable).
```

Different-family separation against 4 third-party apps (Firefox / MacCrab / iTerm / GitHub Desktop), single-Mac smoke:

```
$ maccrabctl mcfp separation /tmp/mcfp-mini-sep.jsonl
MCFP R2 Different-Family Separation
====================================
Corpus:               /tmp/mcfp-mini-sep.jsonl (269 binaries)
Distinct families:    4
Pairs compared:       14747
Pairs cs∨ent differ:  14747  (100.00%)
Pairs fully differ:   13134  ( 89.06%)
95% CI:              [ 99.97%, 100.00%]

R2 separation verdict (CI lower-bound ≥ 80%): PASS
```

Strong single-Mac separation: every one of 14,747 cross-family pairs produces diverging `cs` or `ent`. This is the expected R2-separation shape — different vendors carry different team ids + entitlement sets. The Wilson 95% CI lower-bound of 99.97% is far above the 80% ship gate.

Note: this single-Mac smoke does not substitute for the cross-Mac stability measurement, which still requires Mac #2. It does confirm the *separation* dimension passes against real-world apps; the *stability* dimension is what's still open.
